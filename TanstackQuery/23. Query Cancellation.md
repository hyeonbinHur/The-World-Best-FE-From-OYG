# 23. 쿼리 취소 (Query Cancellation)

## 쿼리 취소란?

TanStack Query는 각 쿼리 함수에 `AbortSignal` 인스턴스를 제공한다. 쿼리가 **오래되었거나(out-of-date)** **비활성 상태(inactive)**가 되면, 이 signal이 abort 된다. 즉, 모든 쿼리는 취소 가능하며, 쿼리 함수 내부에서 취소에 대응할 수 있다. 일반적인 `async/await` 문법을 그대로 사용하면서 자동 취소의 이점을 누릴 수 있다.

> `AbortController` API는 대부분의 런타임 환경에서 사용할 수 있지만, 지원하지 않는 환경에서는 폴리필이 필요하다.

---

## 기본 동작 (Default Behavior)

기본적으로 **컴포넌트가 언마운트되거나 쿼리가 미사용 상태가 되어도**, Promise가 resolve 되기 전에 쿼리가 자동으로 취소되지는 않는다. Promise가 resolve 되면 결과 데이터는 캐시에 저장된다.

이 동작은 유용하다:
- 쿼리를 받기 시작한 후 컴포넌트를 언마운트했다가, 다시 마운트했을 때 가비지 컬렉션이 되지 않았다면 **캐시된 데이터를 바로 사용**할 수 있다.

하지만 `AbortSignal`을 직접 소비(consume)하면 Promise가 취소되고(예: fetch abort), 쿼리도 함께 취소된다. 이 경우 **쿼리 상태는 이전 상태로 되돌아간다(revert)**.

## 사용 예제

### fetch 사용

```tsx
const query = useQuery({
  queryKey: ['todos'],
  queryFn: async ({ signal }) => {
    const todosResponse = await fetch('/todos', {
      signal, // signal을 fetch에 전달
    })
    const todos = await todosResponse.json()

    const todoDetails = todos.map(async ({ details }) => {
      const response = await fetch(details, {
        signal, // 여러 fetch에도 전달 가능
      })
      return response.json()
    })

    return Promise.all(todoDetails)
  },
})
```

### axios v0.22.0+ 사용

```tsx
import axios from 'axios'

const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) =>
    axios.get('/todos', {
      signal, // axios에 signal 전달
    }),
})
```

### axios v0.22.0 미만 사용

```tsx
import axios from 'axios'

const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) => {
    const CancelToken = axios.CancelToken
    const source = CancelToken.source()

    const promise = axios.get('/todos', {
      cancelToken: source.token,
    })

    // TanStack Query가 abort signal을 보내면 요청 취소
    signal?.addEventListener('abort', () => {
      source.cancel('Query was cancelled by TanStack Query')
    })

    return promise
  },
})
```

### XMLHttpRequest 사용

```tsx
const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) => {
    return new Promise((resolve, reject) => {
      var oReq = new XMLHttpRequest()
      oReq.addEventListener('load', () => {
        resolve(JSON.parse(oReq.responseText))
      })
      signal?.addEventListener('abort', () => {
        oReq.abort()
        reject()
      })
      oReq.open('GET', '/todos')
      oReq.send()
    })
  },
})
```

### graphql-request 사용 (v4.0.0+)

```tsx
const client = new GraphQLClient(endpoint)

const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) => {
    client.request({ document: query, signal })
  },
})
```

### graphql-request 사용 (v4.0.0 미만)

```tsx
const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) => {
    const client = new GraphQLClient(endpoint, {
      signal, // 생성자에 signal 전달
    })
    return client.request(query, variables)
  },
})
```

## 수동 취소 (Manual Cancellation)

요청이 오래 걸릴 때, 사용자가 직접 취소 버튼을 눌러 요청을 중단할 수 있도록 하고 싶을 때 사용한다. `queryClient.cancelQueries({ queryKey })`를 호출하면 쿼리가 취소되고 **이전 상태로 되돌아간다**. 쿼리 함수에 전달된 signal을 소비했다면, Promise도 함께 취소된다.

```tsx
const query = useQuery({
  queryKey: ['todos'],
  queryFn: async ({ signal }) => {
    const resp = await fetch('/todos', { signal })
    return resp.json()
  },
})

const queryClient = useQueryClient()

return (
  <button
    onClick={(e) => {
      e.preventDefault()
      queryClient.cancelQueries({ queryKey: ['todos'] })
    }}
  >
    Cancel
  </button>
)
```

## 취소 옵션 (Cancel Options)

쿼리 취소 동작을 세밀하게 제어할 수 있는 옵션이다.

```tsx
// 특정 쿼리를 조용히(silent) 취소
await queryClient.cancelQueries({ queryKey: ['posts'] }, { silent: true })
```

| 옵션 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `silent` | `boolean` | `false` | `true`로 설정하면, `CancelledError`가 observer(예: `onError` 콜백)에 전파되지 않고 관련 알림도 억제된다. retry promise를 반환한다. |
| `revert` | `boolean` | `true` | `true`로 설정하면, 진행 중이던 fetch 직전의 쿼리 상태(data, status)를 복원하고, `fetchStatus`를 `idle`로 설정한다. 이전 데이터가 없을 때만 throw 한다. |

---

## 제한 사항 (Limitations)

Suspense 훅에서는 쿼리 취소가 동작하지 않는다:
- `useSuspenseQuery`
- `useSuspenseQueries`
- `useSuspenseInfiniteQuery`

---

## 요점 정리

| 핵심 개념 | 설명 |
|-----------|------|
| AbortSignal | TanStack Query가 각 쿼리 함수에 자동으로 제공하는 취소 신호 |
| 기본 동작 | 언마운트/미사용 시 Promise를 자동 취소하지 않음 (캐시에 데이터 보존) |
| Signal 소비 시 | fetch가 abort 되고, 쿼리 상태가 이전 상태로 revert |
| 수동 취소 | `queryClient.cancelQueries({ queryKey })`로 직접 취소 가능 |
| silent 옵션 | 취소 에러를 observer에 전파하지 않음 |
| revert 옵션 | 취소 시 이전 상태로 복원 (기본값 `true`) |
| Suspense 제한 | Suspense 훅에서는 쿼리 취소 미지원 |
