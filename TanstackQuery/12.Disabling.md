# 12. 쿼리 비활성화 / 일시 중지 (Disabling / Pausing Queries)

## 쿼리 비활성화란?

쿼리가 자동으로 실행되지 않도록 막고 싶을 때, `enabled = false` 옵션을 사용한다. `enabled`는 boolean을 반환하는 콜백 함수도 받을 수 있다.

## `enabled: false`일 때 동작

| 상황 | 동작 |
|------|------|
| 캐시된 데이터가 있는 경우 | `status === 'success'` (isSuccess) 상태로 초기화 |
| 캐시된 데이터가 없는 경우 | `status === 'pending'`, `fetchStatus === 'idle'` 상태로 시작 |
| 마운트 시 | 자동으로 fetch하지 않음 |
| 백그라운드 | 자동 refetch하지 않음 |
| invalidateQueries / refetchQueries 호출 시 | 무시함 |
| refetch() 수동 호출 | 가능 (단, `skipToken`에서는 불가) |

### 기본 예제

```tsx
function Todos() {
  const { isLoading, isError, data, error, refetch, isFetching } = useQuery({
    queryKey: ['todos'],
    queryFn: fetchTodoList,
    enabled: false, // 자동 실행 비활성화
  })

  return (
    <div>
      <button onClick={() => refetch()}>Fetch Todos</button>

      {data ? (
        <ul>
          {data.map((todo) => (
            <li key={todo.id}>{todo.title}</li>
          ))}
        </ul>
      ) : isError ? (
        <span>Error: {error.message}</span>
      ) : isLoading ? (
        <span>Loading...</span>
      ) : (
        <span>Not ready ...</span>
      )}

      <div>{isFetching ? 'Fetching...' : null}</div>
    </div>
  )
}
```

### 영구 비활성화의 문제점

쿼리를 영구적으로 비활성화하면 TanStack Query의 장점(백그라운드 refetch 등)을 포기하게 된다. 또한 **선언적 방식**(쿼리 의존성을 정의)에서 **명령적 방식**(클릭할 때마다 직접 fetch)으로 바뀌는 것이므로, 대부분의 경우에는 **Lazy Query**가 더 적합하다.

## Lazy Queries (지연 쿼리)

`enabled` 옵션은 영구 비활성화뿐만 아니라, **특정 조건이 충족될 때만 쿼리를 실행**하는 용도로도 사용할 수 있다.

예를 들어, 필터 폼에서 사용자가 필터 값을 입력한 후에만 첫 번째 요청을 보내고 싶은 경우:

```tsx
function Todos() {
  const [filter, setFilter] = React.useState('')

  const { data } = useQuery({
    queryKey: ['todos', filter],
    queryFn: () => fetchTodos(filter),
    enabled: !!filter, // filter가 비어있으면 비활성화
  })

  return (
    <div>
      <FiltersForm onApply={setFilter} />
      {data && <TodosTable data={data} />}
    </div>
  )
}
```

> `filter`가 빈 문자열이면 `!!filter`는 `false` → 쿼리 비활성화  
> `filter`에 값이 들어오면 `!!filter`는 `true` → 쿼리 자동 실행

---

## isLoading vs isPending

Lazy Query는 처음부터 `status: 'pending'` 상태이다. `pending`은 "아직 데이터가 없다"는 의미이기 때문이다. 하지만 쿼리가 비활성화되어 있어 실제로 fetch를 하고 있지 않으므로, 이 상태로 로딩 스피너를 보여주기에는 적절하지 않다.

이럴 때는 **`isLoading`** 플래그를 사용한다:

```
isLoading = isPending && isFetching
```

즉, `isLoading`은 **처음으로 실제 데이터를 가져오고 있을 때만** `true`가 된다.

## skipToken을 활용한 타입 안전한 비활성화

TypeScript 사용 시, `skipToken`을 사용하면 타입 안전하게 쿼리를 비활성화할 수 있다.

```tsx
import { skipToken, useQuery } from '@tanstack/react-query'

function Todos() {
  const [filter, setFilter] = React.useState<string | undefined>()

  const { data } = useQuery({
    queryKey: ['todos', filter],
    queryFn: filter ? () => fetchTodos(filter) : skipToken,
  })

  return (
    <div>
      <FiltersForm onApply={setFilter} />
      {data && <TodosTable data={data} />}
    </div>
  )
}
```

### skipToken vs enabled: false 차이점

| | `enabled: false` | `skipToken` |
|---|---|---|
| `refetch()` 수동 호출 | 가능 | 불가능 (Missing queryFn 에러 발생) |
| 타입 안전성 | 일반적 | queryFn의 타입을 보장 |
| 그 외 동작 | 동일 | 동일 |

> **정리**: 수동으로 `refetch()`를 호출해야 한다면 `enabled: false`를 사용하고, 타입 안전성이 중요하고 수동 refetch가 필요 없다면 `skipToken`을 사용하자.
