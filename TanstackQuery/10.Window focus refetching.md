사용자가 앱을 떠났다가 다시 돌아왔을 때, 쿼리 데이터가 **stale(오래된 상태)**이면 TanStack Query가 자동으로 백그라운드에서 **최신 데이터를 다시 요청**해주는 기능입니다.

`refetchOnWindowFocus` 옵션으로 전역 또는 개별 쿼리 단위로 비활성화 가능합니다.

```tsx
// 전역 비활성화
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      refetchOnWindowFocus: false, // 기본값: true
    },
  },
});

function App() {
  return <QueryClientProvider client={queryClient}>...</QueryClientProvider>;
}
```

```tsx
//개별 비활성화
useQuery({
  queryKey: ["todos"],
  queryFn: fetchTodos,
  refetchOnWindowFocus: false,
});
```

### 커스텀 윈도우 포커스

help)
window focus reteching은 Tanstack Query가 제공하는 전역 싱글톤 객체인 focusManager를 통해 직접 focus를 관리하여 refetching을 커스텀할 수 있습니다.

```tsx
import { focusManager } from "@tanstack/react-query";
```

내부적으로는 TanStack Query 패키지가 로드될 때 자동 생성되고, 기본적으로 브라우저 환경이면 `visibilitychange` 이벤트를 자동으로 구독합니다. SSR/Node 환경에서도 객체 자체는 존재하지만, `window`가 없으므로 이벤트 리스너는 등록되지 않습니다.

help에서 언급한 바와 같이 Tanstack Query는 focus를 관리하기 위해 focusManager라는 전역 객체를 사용하고, visibilitychange라는 이벤트를 이용하여 focus를 감지합니다. focuseManager의 visibilityChange를 바꾸고 싶을 때, 즉 포커스의 기준을 바꾸고 싶을 때 focusManage에 이벤트 리스너를 등록하여 커스텀이 가능합니다.

Tanstack Query에서 정의한 refocus는 focus가 앱 밖으로 나갔다가 다시 들어오는 상황을 의미하지만, 일반적인 브라우저 환경이 아닌, iframe, electron, RN같은 상황에서 커스텀이 필요합니다.

focus manger의 setFocuse를 강제로 true로 만들어주는 방법도 이에 해당합니다.

```tsx
import { focusManager } from "@tanstack/react-query";

// 기본 포커스 상태를 강제로 덮어쓰기
focusManager.setFocused(true);

// 기본 포커스 체크로 되돌리기
focusManager.setFocused(undefined);
```

focus 기준 아예 바꾸기 (iframe)
iframe은 별도의 독립적인 `window`를 가진다. 따라서 탭 간 이동 시 포커스 감지는 가능하지만, **같은 탭 내에서 부모 페이지 ↔ iframe 간 포커스 이동**은 `visibilitychange`로 감지할 수 없습니다. (탭 자체는 안 바뀌니까). 때문에 iframe 내부 별도의 window로 포커스를 감지할 수 있습니다.

```tsx
focusManager.setEventListener((handleFocus) => {
  // iframe의 window에 focus/blur 이벤트를 건다
  const onFocus = () => handleFocus(true);
  const onBlur = () => handleFocus(false);

  window.addEventListener("focus", onFocus);
  window.addEventListener("blur", onBlur);

  return () => {
    window.removeEventListener("focus", onFocus);
    window.removeEventListener("blur", onBlur);
  };
});
```

RN
RN은 window 대신 Appstate 모듈을 사용한다고 합니다. 때문에 app 상태가 'active'||''inactive'인지를 감지하여 refocus 로직을 새로 써 줍니다.

```tsx
import { AppState } from "react-native";
import { focusManager } from "@tanstack/react-query";

function onAppStateChange(status: AppStateStatus) {
  if (Platform.OS !== "web") {
    focusManager.setFocused(status === "active");
  }
}

useEffect(() => {
  const subscription = AppState.addEventListener("change", onAppStateChange);

  return () => subscription.remove();
}, []);
```

window에 이벤트 리스너 등록해준건, clean up 안해주면 계속 쌓이니까 꼭 Clean up 해줘야합니다. 이벤트 리스너 계속 쌓이면, refocus한번에 이벤트 100번 발생합니다.
