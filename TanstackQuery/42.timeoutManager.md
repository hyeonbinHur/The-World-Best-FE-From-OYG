# TimeoutManager

## 핵심 요약

TimeoutManager는 TanStack Query에서 `setTimeout`과 `setInterval` 타이머를 관리하는 매니저입니다. 쿼리의 staleTime, gcTime, 재시도, 스로틀링, 디바운싱 등의 기능을 구현하는 데 사용됩니다.

**왜 필요한가?**
- 수천 개의 쿼리를 다룰 때 이벤트 루프 성능 문제가 발생할 수 있음
- 기본 setTimeout의 최대 지연 시간(약 24일) 제한을 우회하고 싶을 때
- 타이머 성능을 최적화하고 싶을 때

---

## 주요 개념

### 1. 기본 동작
- **기본값**: 전역 `setTimeout`과 `setInterval` 사용
- **커스텀 가능**: 사용자 정의 타이머 구현체로 교체 가능

### 2. 사용 가능한 메서드

```typescript
// 타이머 설정
timeoutManager.setTimeout(callback, delayMs)
timeoutManager.setInterval(callback, intervalMs)

// 타이머 취소
timeoutManager.clearTimeout(timeoutId)
timeoutManager.clearInterval(intervalId)

// 커스텀 Provider 설정
timeoutManager.setTimeoutProvider(customProvider)
```

## 상세 설명

### setTimeoutProvider - 커스텀 타이머 구현체 설정

**언제 사용하나?**
- 수천 개의 쿼리로 인한 성능 문제가 있을 때
- 24일 이상의 긴 타이머가 필요할 때

**중요 주의사항:**
⚠️ `QueryClient`나 쿼리를 생성하기 **전에** 호출해야 합니다. 그래야 애플리케이션의 모든 타이머가 동일한 Provider를 사용합니다. (서로 다른 Provider는 서로의 타이머를 취소할 수 없음)

**타입 정의:**

```typescript
type ManagedTimerId = number | { [Symbol.toPrimitive]: () => number }

type TimeoutProvider<TTimerId extends ManagedTimerId = ManagedTimerId> = {
  readonly setTimeout: (callback: TimeoutCallback, delay: number) => TTimerId
  readonly clearTimeout: (timeoutId: TTimerId | undefined) => void
  readonly setInterval: (callback: TimeoutCallback, delay: number) => TTimerId
  readonly clearInterval: (intervalId: TTimerId | undefined) => void
}
```

**사용 예시:**

```typescript
import { timeoutManager, QueryClient } from '@tanstack/react-query'
import { CustomTimeoutProvider } from './CustomTimeoutProvider'

// 1. 먼저 커스텀 Provider 설정
timeoutManager.setTimeoutProvider(new CustomTimeoutProvider())

// 2. 그 다음 QueryClient 생성
export const queryClient = new QueryClient()
```

### TimeoutProvider 구현 시 고려사항

**성능 최적화 전략:**

1. **단기 타이머 (5초 미만)**
   - 지연 시간에 민감함
   - 빠른 실행이 중요

2. **장기 타이머**
   - 타이머 병합(coalescing)이 유리함
   - 비슷한 시점의 타이머들을 그룹화
   - 계층적 타임 휠(hierarchical time wheel) 같은 자료구조 활용 가능

**Timer ID 처리:**
- NodeJS는 `setTimeout`/`setInterval`에서 객체를 반환함
- `Symbol.toPrimitive`를 통해 number로 변환 가능한 객체여야 함
- 구현체는 내부적으로 number로 변환하거나, 자체 커스텀 객체 타입 반환 가능

---

### 기본 메서드 사용법

#### setTimeout

전역 `setTimeout`처럼 동작합니다.

```typescript
import { timeoutManager } from '@tanstack/react-query'

const timeoutId = timeoutManager.setTimeout(
  () => console.log('ran at:', new Date()),
  1000, // 1초 후 실행
)

// 타이머 취소
timeoutManager.clearTimeout(timeoutId)

// Timer ID를 number로 변환 가능
const timeoutIdNumber: number = Number(timeoutId)
```

#### setInterval

전역 `setInterval`처럼 동작합니다.

```typescript
import { timeoutManager } from '@tanstack/react-query'

const intervalId = timeoutManager.setInterval(
  () => console.log('ran at:', new Date()),
  1000, // 1초마다 반복 실행
)

// 인터벌 취소
timeoutManager.clearInterval(intervalId)
```

## 어려운 개념 풀이

### 1. Symbol.toPrimitive란?

객체를 원시 타입(number, string 등)으로 변환할 때 호출되는 특수 메서드입니다.

```typescript
const timerObj = {
  [Symbol.toPrimitive]() {
    return 123 // number로 변환될 때 123 반환
  }
}

Number(timerObj) // 123
```

**왜 필요한가?**
- NodeJS의 타이머는 객체를 반환하지만, TanStack Query는 내부적으로 number로 관리하고 싶을 수 있음
- 이 인터페이스를 통해 두 가지 방식을 모두 지원

### 2. 타이머 병합(Timer Coalescing)이란?

여러 타이머를 효율적으로 관리하는 기법입니다.

**예시:**
```
타이머 A: 10:00:05에 실행
타이머 B: 10:00:06에 실행
타이머 C: 10:00:07에 실행

→ 병합: 10:00:07에 모두 함께 실행
```

**장점:**
- 이벤트 루프 부하 감소
- CPU 웨이크업 횟수 감소로 배터리 절약 (모바일)
- 장기 타이머에 특히 유용

### 3. 계층적 타임 휠(Hierarchical Time Wheel)이란?

효율적인 타이머 관리를 위한 자료구조입니다.

**일반적인 방법:**
- 배열에 모든 타이머 저장 → O(n) 시간 복잡도로 다음 타이머 찾기

**타임 휠 방법:**
- 시계처럼 원형 버킷에 타이머 분산
- O(1) 시간으로 다음 타이머 찾기
- 계층 구조로 단기/장기 타이머를 다른 "바퀴"에 배치

## 실전 활용 시나리오

### 시나리오 1: 성능 문제 해결

```typescript
// 문제: 1000개의 쿼리가 각각 타이머를 가지면 성능 저하
// 해결: 커스텀 Provider로 타이머 병합

import { timeoutManager } from '@tanstack/react-query'
import { CoalescingTimeoutProvider } from './providers'

timeoutManager.setTimeoutProvider(new CoalescingTimeoutProvider({
  coalescingWindow: 100, // 100ms 내의 타이머는 병합
}))
```

### 시나리오 2: 장기 타이머 지원

```typescript
// 문제: 30일 후 캐시 만료가 필요한데 기본 setTimeout은 24일까지만 지원
// 해결: 긴 지연 시간을 지원하는 커스텀 Provider

import { LongDelayTimeoutProvider } from './providers'

timeoutManager.setTimeoutProvider(new LongDelayTimeoutProvider())
```

## 핵심 정리

1. **TimeoutManager = TanStack Query의 타이머 관리자**
   - 기본적으로 브라우저/NodeJS 타이머 사용
   - 커스텀 구현으로 교체 가능

2. **언제 커스터마이징?**
   - 대량의 쿼리로 성능 문제 발생 시
   - 24일 이상의 긴 타이머 필요 시
   - 타이머 병합으로 최적화 하고 싶을 때

3. **주의사항**
   - QueryClient 생성 전에 setTimeoutProvider 호출
   - Timer ID는 number 또는 Symbol.toPrimitive 구현 객체

4. **성능 최적화 기법**
   - 단기 타이머: 낮은 지연 시간 중요
   - 장기 타이머: 타이머 병합으로 효율성 향상
   - 계층적 타임 휠로 대량 타이머 관리
