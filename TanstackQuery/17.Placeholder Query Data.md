# 17. Placeholder Query Data (플레이스홀더 쿼리 데이터)

## Placeholder Data란?

Placeholder Data는 쿼리가 이미 데이터를 가지고 있는 것처럼 동작하게 해준다. `initialData`와 비슷하지만, **캐시에 저장되지 않는다**는 점이 핵심 차이이다.

실제 데이터를 백그라운드에서 가져오는 동안, 부분적이거나 임시적인 데이터를 사용해 UI를 빠르게 렌더링할 수 있다.

### 예시 상황

블로그 게시글 목록에서 제목과 본문 미리보기만 포함된 데이터를 가지고 있다고 하자. 개별 게시글 상세 쿼리의 placeholder로 이 데이터를 활용하면, 실제 전체 데이터를 가져오는 동안에도 콘텐츠 레이아웃을 즉시 보여줄 수 있다.

### Placeholder Data 제공 방식

| 방식 | 설명 |
|------|------|
| 선언적 (Declaratively) | `useQuery`의 `placeholderData` 옵션에 직접 제공 |
| 명령적 (Imperatively) | `queryClient`의 `prefetch` 또는 `fetch`와 함께 `placeholderData` 옵션 사용 |

### Placeholder Data 사용 시 쿼리 상태

- `pending` 상태가 아니라 **`success` 상태**로 시작한다 (표시할 데이터가 있으므로).
- 실제 데이터와 구분하기 위해 쿼리 결과에 **`isPlaceholderData: true`** 플래그가 설정된다.

---

## 1. 값으로 제공하기 (Placeholder Data as a Value)

가장 간단한 방식으로, 정적인 placeholder 값을 직접 전달한다.

```tsx
function Todos() {
  const result = useQuery({
    queryKey: ['todos'],
    queryFn: () => fetch('/todos'),
    placeholderData: placeholderTodos,
  })
}
```

---

## 2. 메모이제이션 (Placeholder Data Memoization)

Placeholder 데이터를 생성하는 과정이 비용이 큰 경우, `useMemo`를 사용해 매 렌더링마다 재생성되지 않도록 메모이제이션할 수 있다.

```tsx
function Todos() {
  const placeholderData = useMemo(() => generateFakeTodos(), [])
  const result = useQuery({
    queryKey: ['todos'],
    queryFn: () => fetch('/todos'),
    placeholderData,
  })
}
```

---

## 3. 함수로 제공하기 (Placeholder Data as a Function)

`placeholderData`에 함수를 전달하면, **이전에 성공한 쿼리의 데이터와 메타 정보**에 접근할 수 있다.

QueryKey가 변경될 때(예: `['todos', 1]` -> `['todos', 2]`), 로딩 스피너 대신 **이전 데이터를 계속 표시**할 수 있어 페이지네이션 등에서 유용하다.

```tsx
const result = useQuery({
  queryKey: ['todos', id],
  queryFn: () => fetch(`/todos/${id}`),
  placeholderData: (previousData, previousQuery) => previousData,
})
```

> `previousData`: 직전 성공 쿼리의 데이터  
> `previousQuery`: 직전 성공 쿼리의 메타 정보

---

## 4. 캐시에서 가져오기 (Placeholder Data from Cache)

다른 쿼리의 캐시 데이터를 placeholder로 활용할 수 있다. 예를 들어, 블로그 게시글 목록 캐시에서 특정 게시글의 미리보기 데이터를 꺼내 개별 게시글 쿼리의 placeholder로 사용할 수 있다.

```tsx
function BlogPost({ blogPostId }) {
  const queryClient = useQueryClient()
  const result = useQuery({
    queryKey: ['blogPost', blogPostId],
    queryFn: () => fetch(`/blogPosts/${blogPostId}`),
    placeholderData: () => {
      return queryClient
        .getQueryData(['blogPosts'])
        ?.find((d) => d.id === blogPostId)
    },
  })
}
```

---

## Placeholder Data vs Initial Data 비교

| | `placeholderData` | `initialData` |
|---|---|---|
| 캐시 저장 | X (캐시에 저장되지 않음) | O (캐시에 저장됨) |
| 초기 상태 | `success` + `isPlaceholderData: true` | `success` |
| 실제 데이터 도착 시 | placeholder가 실제 데이터로 교체됨 | `staleTime` 이후 refetch |
| 용도 | 빠른 UI 렌더링용 임시 데이터 | 실제 초기값으로 사용할 수 있는 데이터 |

> Placeholder Data와 Initial Data의 상세 비교는 TkDodo의 아티클을 참고하자.
