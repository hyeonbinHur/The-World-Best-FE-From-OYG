- Queries란
  - useQuery, useMutation: Get, Post를 포함한 Promise기반 선언적 메서드를 고유키를 이용하여 선언적 의존성을 주입하는 과정이다.
  - useQuery, useMutation이 Promise기반 메서드를 호출하는 역활도 있지만, 고유 키와 Promise 기반 메서드를 묶어주는 역활이 중점이다.
- Query Result
  - 전통적인 라이브러리는 "로딩 중" 하나의 상태를 사졌지만, Tanstack Query는 캐시된 데이터와, 최신화 상태를 표현하고 싶어 상태를 status, fetchStatus 두개로 나누었다.
  - status: 데이터가 있니..? -> 상태 중심
    - pending, error, success,
    - 파생 boolean
      - isPending, isError, isSuccess
        - isPending: Promise 실행중이고, 캐싱된 데이터도 없다면 true이다. 캐싱된 데이터가 있으면 false기 때문에 스피너 안보여줘도 된다.
        - isError: 실패인지 성공인지
        - isSuccess: 실패인지 성공인지
        - error: isError가 true라면 promise error 객체가 들어간다.
        - data: promise가 성공일때 결과가 들어간다.
  - fetchStatus: 함수가 동작중이니..?: -> 동작 중심
    - fetching, paused, idle
    - 파생 boolean
      - isFetching, isPaused
      - isFetching: 캐싱된 데이터가 있든 없든 Promise가 실행중이면 true이다.
      - isPaused: 네트워크가 끊기는 등 데이터 호출이 멈춘 상태.
