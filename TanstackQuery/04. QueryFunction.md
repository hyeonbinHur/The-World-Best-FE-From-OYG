Query Fn는 Promise를 반환하는 모든 함수를 의미한다: aka resolve, error throw, pending을 반환해야한다.

axios같은 promise함수는 기본적으로 throw error 로직이 존재하지만, fetch 함수는 error를 throw하지 않기 때문에 명시적으로 error를 throw해주거나 promise reject를 해줘야 Queries에서 error 상태 관리가 가능하다.

- QueryFunctionContext
  - Queries에서 Promise 함수를 QueryFn으로 사용할 경우, 사용한 Promise함수에 자동으로 QueryFunctionContext를 전달하여 사용한다.
  - QueryFunctionContext에는 쿼리키, QueryClient, Signal, meta 데이터, pageParam이 포함된다.
  - - pageParam의 경우 QueryFunctionContext에 있지만 useQuery에서는 undefined이고, useInfinityQuery를 쓸 때 보다 의미있는 정보가 포함된다.

```ts
function Todos({ status, page }) {
  const result = useQuery({
    queryKey: ["todos", { status, page }],
    queryFn: fetchTodoList,
  });

  return result;
}

// 쿼리 함수 내부에서 key, status, page 변수에 접근하세요!
function fetchTodoList({ queryKey }) {
  const [_key, { status, page }] = queryKey;
  console.log(meta);
  console.log(signal);
  return new Promise((resolve, reject) => {
    // ...
  });
}
```

- meta의 경우 별도의 추가적인 정보를 저장하기 위해 사용되며, 일반적으로 에러 메시지 같은 값을 저장합니다.

```ts
const result = useQuery({
  queryKey: ["todos"],
  queryFn: fetchTodos,
  meta: {
    errorMessage: "할 일 목록을 불러오지 못했습니다",
    source: "dashboard",
  },
});
```
