Query를 동시에 실행하는 것을 의미하며 데이터 동시성을 극대화하기 위해 사용된다.

### Manual Parallel Queries

useQuery와 useInfiniteQuery를 나란히(code line이 아닌 render cycle) 사용하면 별도의 설정 필요 없이 병렬적으로 수행된다.

```tsx
function App () {
  // 아래 쿼리들은 병렬로 실행된다
  const usersQuery = useQuery({ queryKey: ['users'], queryFn: fetchUsers })
  const teamsQuery = useQuery({ queryKey: ['teams'], queryFn: fetchTeams })
  const projectsQuery = useQuery({ queryKey: ['projects'], queryFn: fetchProjects })
  ...
}
```

---

### Dynamic Parallel Queries

렌더링마다 실행해야 할 쿼리가 동적으로 변하는 경우, 수동 쿼리 방식을 사용할 수 없기 때문에 (애초에 runtime에 결정되기 때문에 코드 라인에 표시가 될 수 없습니다), TanstackQuery는 useQuries라는 훅을 제공합니다.

```tsx
function App({ users }) {
  const userQueries = useQueries({
    queries: users.map((user) => {
      return {
        queryKey: ["user", user.id],
        queryFn: () => fetchUserById(user.id),
      };
    }),
  });
}
```

Suspense에서의 병렬 쿼리 주의

_Suspense는 fetching이 완료되기 전까지 fallback을 보여주는 모드를 말합니당_

```tsx
<Suspense fallback={<Loading />}>
  <TodoList />
</Suspense>
```

Suspense로 감싸진 컴포넌트는 Manual Parallel이 동작하지 않습니다.

- 내부적으로 Promise를 throw하여 컴포넌트를 suspende시키기 때문에 useSuspenseQueries라는 별도의 훅을 이용해주어야 합니다.

useSuspens와 useQuery의 차이점

useQuery는 데이터가 없어도, 즉 fetching중일때도 isPending, isFetching 등등 로딩 상태를 리턴하지만 useSuspenseQuery는 Promise를 Throw합니다. 때문에 useSuspenseQuery 아래 코드가 아예 실행이 되지 않습니다!

```tsx
function UserProfile() {
  console.log("1. 쿼리 호출 전");

  const { data } = useSuspenseQuery({ queryKey: ["user"], queryFn: fetchUser });

  // 데이터가 없으면 위에서 throw → 아래 코드 전부 실행 안 됨
  console.log("2. 쿼리 호출 후"); // ← 로딩 중엔 안 찍힘
  return <div>{data.name}</div>; // ← 로딩 중엔 실행 안 됨
}
```
