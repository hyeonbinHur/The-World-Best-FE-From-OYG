# Render Optimizations (렌더링 최적화)

## 개요
React Query는 컴포넌트가 실제로 필요할 때만 리렌더링되도록 여러 최적화를 자동으로 적용합니다.

## 주요 최적화 기법

### 1. Structural Sharing (구조적 공유)
**개념**: 리렌더링 간 가능한 많은 참조를 유지하는 기법

**작동 방식**:
- 네트워크로 데이터를 가져올 때 JSON 파싱으로 완전히 새로운 참조가 생성됨
- 데이터에 변경이 없으면 원본 참조를 유지
- 일부만 변경된 경우 변경되지 않은 부분은 유지하고 변경된 부분만 교체

**설정**:
```javascript
// 전역 또는 쿼리별로 비활성화
structuralSharing: false

// 또는 커스텀 구조적 공유 함수 제공
structuralSharing: (oldData, newData) => { ... }
```

**참고**: JSON 호환 데이터를 반환하는 queryFn에서만 작동

### 2. Referential Identity (참조 동일성)
**특징**:
- `useQuery`, `useInfiniteQuery`, `useMutation`에서 반환되는 최상위 객체는 **매 렌더링마다 새로운 참조**
- 하지만 반환되는 `data` 속성은 **가능한 한 안정적으로 유지**됨

### 3. Tracked Properties (추적된 속성)
**개념**: 실제로 사용되는 속성이 변경될 때만 리렌더링 트리거

**작동 방식**:
- Proxy 객체를 사용하여 구현
- `isFetching`, `isStale` 같은 속성이 자주 변경되어도, 컴포넌트에서 사용하지 않으면 리렌더링 안 됨

**설정**:
```javascript
// 전역 또는 쿼리별로 커스터마이징
notifyOnChangeProps: ['data', 'error']

// 기능 비활성화 (모든 속성 변경 시 리렌더링)
notifyOnChangeProps: 'all'
```

**주의사항**:
- 객체 rest 구조 분해를 사용하면 이 최적화가 비활성화됨
- Proxy의 get trap은 구조 분해 또는 직접 접근으로 속성에 접근할 때 호출됨
- 린트 규칙으로 이러한 함정 방지 가능

### 4. Select (선택 옵션)
**개념**: 컴포넌트가 구독할 데이터의 일부만 선택

**장점**:
- 고도로 최적화된 데이터 변환
- 불필요한 리렌더링 방지

**예제**:
```javascript
export const useTodos = (select) => {
  return useQuery({
    queryKey: ['todos'],
    queryFn: fetchTodos,
    select,
  })
}

export const useTodoCount = () => {
  return useTodos((data) => data.length)
}
```

**결과**:
- `useTodoCount`를 사용하는 컴포넌트는 todos의 길이가 변경될 때만 리렌더링
- todo의 name이 변경되어도 리렌더링되지 않음

**주의사항**:
- `select`는 성공적으로 캐시된 데이터에 대해서만 작동
- 에러를 던지는 적절한 장소가 아님
- 에러의 출처는 `queryFn`이어야 함
- `select`에서 에러 반환 시 `data`는 `undefined`, `isSuccess`는 `true`가 됨

### 5. Memoization (메모이제이션)
**select 함수 재실행 조건**:
1. select 함수 자체가 참조적으로 변경됨
2. data가 변경됨

**문제점**:
- 인라인 select 함수는 매 렌더링마다 실행됨

**해결 방법**:

1. **useCallback 사용**:
```javascript
export const useTodoCount = () => {
  return useTodos(useCallback((data) => data.length, []))
}
```

2. **안정적인 함수 참조로 추출**:
```javascript
const selectTodoCount = (data) => data.length

export const useTodoCount = () => {
  return useTodos(selectTodoCount)
}
```

## 핵심 요약
1. **Structural Sharing**: 변경되지 않은 데이터 참조 유지
2. **Referential Identity**: data 속성은 안정적, 최상위 객체는 매번 새로운 참조
3. **Tracked Properties**: 사용되는 속성만 추적하여 리렌더링 최소화
4. **Select**: 필요한 데이터만 구독하여 최적화
5. **Memoization**: select 함수를 메모이제이션하여 불필요한 재실행 방지
