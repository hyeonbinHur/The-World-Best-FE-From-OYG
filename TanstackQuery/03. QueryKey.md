Query Key는 Promise 기반 결과를 캐싱, 맵핑해주는 것에 목적이 있다.

반드시 배열 형태여야하고, 단순한 문자열로 우리우진 형태부터, 객체 배열까지 가능하다. JSON 직렬화가 가능하고 쿼리 데이터에 대해 Unique하다면 무엇이든 가능하다.

- Simple Query Keys
  - 계층 구조가 없는, 상수로 이루어진 형태

```TS
	useQuery({ queryKey: ['something', 'special'], ... })
```

- Array Keys with variables
  - 고유 데이터를 식별하기 위해 더 고유한 정보가 필요한 경우 (id), 객체를 배열에 담아 사용한다.
  - 중첩 리소스, 추가 매개변수가 있는 쿼리.

```TS
// 개별 할 일 항목 (ID 포함)
useQuery({ queryKey: ['todo', 5], ... })

// "미리보기(preview)" 형식의 개별 할 일 항목
useQuery({ queryKey: ['todo', 5, { preview: true }], ...})

// "완료된(done)" 상태의 할 일 목록
useQuery({ queryKey: ['todos', { type: 'done' }], ... })
```

- 결정론적 캐싱
  - 배열 그자체의 index는 고유 키로 간주된다,
    - key: ['todo', 1]과 [1, 'todo']는 다른 키로 간주된다
    - key: ['todo, {status: 'active', page: 1}], ['todo', {page:1, status: 'active'}]는 같은 키로 간주된다.
      - 객체에는 index가 없기 때문에, 순서보다는 직렬화 하여 내부 속성 값을 중점으로 본다.

- 변수를 키에 포함하는 경우
  - 쿼리 키는 페칭하려는 데이터를 고유하게 맵핑하므로, 쿼리 함수 내에서 사용되는 "변경 가능한 모든 변수"를 쿼리 키에 포함해야 정확하게 사용 가능합니다.

```TS
function Todos({ todoId }) {
	const result = useQuery({
		queryKey: ['todos', todoId],
		queryFn: () => fetchTodoById(todoId),
	})
}
```
