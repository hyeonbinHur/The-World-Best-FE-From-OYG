# 18. Mutations (뮤테이션)

## Mutation이란?

쿼리(Query)가 데이터를 **조회**하는 데 사용된다면, 뮤테이션(Mutation)은 데이터를 **생성/수정/삭제**하거나 서버 사이드 이펙트를 수행하는 데 사용된다. TanStack Query에서는 `useMutation` 훅을 제공한다.

### 기본 예제

```tsx
function App() {
  const mutation = useMutation({
    mutationFn: (newTodo) => {
      return axios.post('/todos', newTodo)
    },
  })

  return (
    <div>
      {mutation.isPending ? (
        'Adding todo...'
      ) : (
        <>
          {mutation.isError ? (
            <div>An error occurred: {mutation.error.message}</div>
          ) : null}

          {mutation.isSuccess ? <div>Todo added!</div> : null}

          <button
            onClick={() => {
              mutation.mutate({ id: new Date(), title: 'Do Laundry' })
            }}
          >
            Create Todo
          </button>
        </>
      )}
    </div>
  )
}
```

---

## Mutation의 상태

뮤테이션은 항상 다음 네 가지 상태 중 하나에 있다:

| 상태 | 플래그 | 설명 |
|------|--------|------|
| idle | `isIdle` / `status === 'idle'` | 초기 상태이거나 리셋된 상태 |
| pending | `isPending` / `status === 'pending'` | 현재 실행 중인 상태 |
| error | `isError` / `status === 'error'` | 에러가 발생한 상태 |
| success | `isSuccess` / `status === 'success'` | 성공적으로 완료된 상태 |

### 상태별 사용 가능한 추가 정보

- **error 상태**: `mutation.error`로 에러 객체에 접근 가능
- **success 상태**: `mutation.data`로 응답 데이터에 접근 가능

## mutate 함수에 변수 전달

`mutate` 함수에 단일 변수 또는 객체를 전달하여 `mutationFn`에 인자를 넘길 수 있다.

> **주의 (React 16 이하)**: `mutate`는 비동기 함수이므로 React 16 이하에서는 이벤트 콜백에서 직접 사용할 수 없다. React의 이벤트 풀링 때문에 별도의 함수로 감싸야 한다.

```tsx
// React 16 이하에서는 이렇게 사용해야 한다
const CreateTodo = () => {
  const mutation = useMutation({
    mutationFn: (formData) => {
      return fetch('/api', formData)
    },
  })
  const onSubmit = (event) => {
    event.preventDefault()
    mutation.mutate(new FormData(event.target))
  }

  return <form onSubmit={onSubmit}>...</form>
}
```

## Mutation 상태 리셋 (Resetting Mutation State)

뮤테이션의 에러나 데이터를 초기화하고 싶을 때 `reset()` 함수를 사용한다.

```tsx
const CreateTodo = () => {
  const [title, setTitle] = useState('')
  const mutation = useMutation({ mutationFn: createTodo })

  const onCreateTodo = (e) => {
    e.preventDefault()
    mutation.mutate({ title })
  }

  return (
    <form onSubmit={onCreateTodo}>
      {mutation.error && (
        <h5 onClick={() => mutation.reset()}>{mutation.error}</h5>
      )}
      <input
        type="text"
        value={title}
        onChange={(e) => setTitle(e.target.value)}
      />
      <br />
      <button type="submit">Create Todo</button>
    </form>
  )
}
```

## Mutation 사이드 이펙트 (Side Effects)

`useMutation`은 뮤테이션 생명주기의 각 단계에서 실행되는 콜백 옵션을 제공한다. 쿼리 무효화(invalidation), refetch, 낙관적 업데이트 등에 매우 유용하다.

| 콜백 | 실행 시점 | 매개변수 |
|------|-----------|----------|
| `onMutate` | mutate 호출 직후, mutationFn 실행 전 | `(variables, context)` |
| `onError` | 에러 발생 시 | `(error, variables, onMutateResult, context)` |
| `onSuccess` | 성공 시 | `(data, variables, onMutateResult, context)` |
| `onSettled` | 성공이든 에러든 완료 시 | `(data, error, variables, onMutateResult, context)` |

### 사이드 이펙트 예제

```tsx
useMutation({
  mutationFn: addTodo,
  onMutate: (variables, context) => {
    // 뮤테이션 실행 직전
    // 롤백에 사용할 데이터를 반환할 수 있음
    return { id: 1 }
  },
  onError: (error, variables, onMutateResult, context) => {
    // 에러 발생 시 롤백
    console.log(`rolling back optimistic update with id ${onMutateResult.id}`)
  },
  onSuccess: (data, variables, onMutateResult, context) => {
    // 성공!
  },
  onSettled: (data, error, variables, onMutateResult, context) => {
    // 성공이든 에러든 항상 실행
  },
})
```

### 콜백에서 Promise 반환

콜백 함수에서 Promise를 반환하면, 해당 Promise가 완료될 때까지 다음 콜백 실행이 대기된다.

```tsx
useMutation({
  mutationFn: addTodo,
  onSuccess: async () => {
    console.log("I'm first!")
  },
  onSettled: async () => {
    console.log("I'm second!")
  },
})
```

### mutate 호출 시 추가 콜백

`mutate()` 함수에도 `onSuccess`, `onError`, `onSettled` 콜백을 전달할 수 있다. 이 콜백은 `useMutation`에 정의된 콜백 **이후에** 실행된다. 컴포넌트별 사이드 이펙트를 처리할 때 유용하다.

> **주의**: 컴포넌트가 뮤테이션 완료 전에 언마운트되면, `mutate`에 전달한 추가 콜백은 실행되지 않는다.

```tsx
useMutation({
  mutationFn: addTodo,
  onSuccess: (data, variables, onMutateResult, context) => {
    // 먼저 실행됨
  },
})

mutate(todo, {
  onSuccess: (data, variables, onMutateResult, context) => {
    // 나중에 실행됨
  },
})
```

## 연속 뮤테이션 (Consecutive Mutations)

연속으로 `mutate()`를 호출할 때의 콜백 동작 차이:

| 위치 | 동작 |
|------|------|
| `useMutation`의 콜백 | 매 `mutate` 호출마다 실행됨 |
| `mutate()`의 콜백 | 마지막 `mutate` 호출에 대해서만 1번 실행됨 (컴포넌트가 마운트되어 있을 때) |

```tsx
useMutation({
  mutationFn: addTodo,
  onSuccess: (data, variables, onMutateResult, context) => {
    // 3번 호출됨
  },
})

const todos = ['Todo 1', 'Todo 2', 'Todo 3']
todos.forEach((todo) => {
  mutate(todo, {
    onSuccess: (data, variables, onMutateResult, context) => {
      // 마지막 뮤테이션(Todo 3)에 대해서만 1번 실행됨
      // 어떤 뮤테이션이 먼저 resolve되든 상관없이
    },
  })
})
```

> **주의**: `mutationFn`은 비동기이므로, 뮤테이션이 완료되는 순서가 `mutate` 호출 순서와 다를 수 있다.

## Promises (mutateAsync)

`mutate` 대신 `mutateAsync`를 사용하면 Promise를 반환받아 `async/await`으로 처리할 수 있다.

```tsx
const mutation = useMutation({ mutationFn: addTodo })

try {
  const todo = await mutation.mutateAsync(todo)
  console.log(todo)
} catch (error) {
  console.error(error)
} finally {
  console.log('done')
}
```

## 재시도 (Retry)

기본적으로 TanStack Query는 뮤테이션 에러 시 재시도하지 않는다. `retry` 옵션으로 재시도 횟수를 설정할 수 있다.

```tsx
const mutation = useMutation({
  mutationFn: addTodo,
  retry: 3,
})
```

- 디바이스가 오프라인으로 인해 실패한 뮤테이션은, 다시 온라인이 되면 같은 순서로 재시도된다.

## 뮤테이션 영속화 (Persist Mutations)

뮤테이션을 스토리지에 저장해두었다가 나중에 재개할 수 있다. hydration 함수를 사용한다.

### 핵심 흐름

1. `queryClient.setMutationDefaults`로 뮤테이션 기본값을 정의
2. `dehydrate(queryClient)`로 상태를 직렬화
3. `hydrate(queryClient, state)`로 상태를 복원
4. `queryClient.resumePausedMutations()`로 일시 중지된 뮤테이션 재개

```tsx
const queryClient = new QueryClient()

queryClient.setMutationDefaults(['addTodo'], {
  mutationFn: addTodo,
  onMutate: async (variables, context) => {
    await context.client.cancelQueries({ queryKey: ['todos'] })
    const optimisticTodo = { id: uuid(), title: variables.title }
    context.client.setQueryData(['todos'], (old) => [...old, optimisticTodo])
    return { optimisticTodo }
  },
  onSuccess: (result, variables, onMutateResult, context) => {
    context.client.setQueryData(['todos'], (old) =>
      old.map((todo) =>
        todo.id === onMutateResult.optimisticTodo.id ? result : todo,
      ),
    )
  },
  onError: (error, variables, onMutateResult, context) => {
    context.client.setQueryData(['todos'], (old) =>
      old.filter((todo) => todo.id !== onMutateResult.optimisticTodo.id),
    )
  },
  retry: 3,
})

// 뮤테이션 실행
const mutation = useMutation({ mutationKey: ['addTodo'] })
mutation.mutate({ title: 'title' })

// 앱 종료 시 상태 저장
const state = dehydrate(queryClient)

// 앱 시작 시 상태 복원
hydrate(queryClient, state)

// 일시 중지된 뮤테이션 재개
queryClient.resumePausedMutations()
```

## 오프라인 뮤테이션 영속화 (Persisting Offline Mutations)

`persistQueryClient` 플러그인으로 오프라인 뮤테이션을 영속화할 때, **반드시 기본 뮤테이션 함수를 제공해야 한다**.

> **이유**: 함수는 직렬화할 수 없으므로, 상태만 저장된다. 페이지 리로드 후 `resumePausedMutations()`를 호출하면 `mutationFn`이 없어서 `No mutationFn found` 에러가 발생한다.

```tsx
const persister = createSyncStoragePersister({
  storage: window.localStorage,
})
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      gcTime: 1000 * 60 * 60 * 24, // 24시간
    },
  },
})

// 페이지 리로드 후에도 뮤테이션이 재개될 수 있도록 기본 함수 설정
queryClient.setMutationDefaults(['todos'], {
  mutationFn: ({ id, data }) => {
    return api.updateTodo(id, data)
  },
})

export default function App() {
  return (
    <PersistQueryClientProvider
      client={queryClient}
      persistOptions={{ persister }}
      onSuccess={() => {
        // localStorage에서 복원 성공 후 뮤테이션 재개
        queryClient.resumePausedMutations()
      }}
    >
      <RestOfTheApp />
    </PersistQueryClientProvider>
  )
}
```

## Mutation Scopes (뮤테이션 스코프)

기본적으로 모든 뮤테이션은 **병렬**로 실행된다. 같은 뮤테이션의 `.mutate()`를 여러 번 호출해도 마찬가지이다.

`scope`를 지정하면 같은 `scope.id`를 가진 뮤테이션들이 **직렬**로 실행된다. 이미 같은 스코프의 뮤테이션이 진행 중이면 `isPaused: true` 상태로 큐에 대기하다가 순서가 오면 자동 실행된다.

```tsx
const mutation = useMutation({
  mutationFn: addTodo,
  scope: {
    id: 'todo',
  },
})
```

## 핵심 요약

| 개념 | 설명 |
|------|------|
| `useMutation` | 데이터 생성/수정/삭제를 위한 훅 |
| `mutate()` | 뮤테이션 실행 (fire-and-forget) |
| `mutateAsync()` | Promise를 반환하는 뮤테이션 실행 |
| `reset()` | 뮤테이션 상태(에러/데이터)를 초기화 |
| 사이드 이펙트 콜백 | `onMutate` → `onSuccess`/`onError` → `onSettled` 순서로 실행 |
| 연속 뮤테이션 | `useMutation` 콜백은 매번, `mutate` 콜백은 마지막만 실행 |
| retry | 기본값은 재시도 없음, `retry` 옵션으로 설정 가능 |
| 영속화 | `dehydrate`/`hydrate`로 뮤테이션 상태를 저장/복원 가능 |
| scope | 같은 `scope.id`의 뮤테이션을 직렬로 실행 |
