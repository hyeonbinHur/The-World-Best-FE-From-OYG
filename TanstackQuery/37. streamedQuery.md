# streamedQuery

## 핵심 개념

`streamedQuery`는 스트리밍 데이터를 처리하기 위한 헬퍼 함수로, AsyncIterable에서 데이터를 조금씩 받아서 쿼리 함수를 생성합니다.

**실험적 기능**: 현재 커뮤니티 피드백을 수집 중인 실험적 API입니다.

## 기본 사용법

```tsx
import { experimental_streamedQuery as streamedQuery } from '@tanstack/react-query'

const query = queryOptions({
  queryKey: ['data'],
  queryFn: streamedQuery({
    streamFn: fetchDataInChunks,
  }),
})
```

## 동작 방식 요약

1. **데이터 형태**: 받은 모든 청크(chunk)들을 배열로 저장
2. **상태 변화**:
   - 첫 청크를 받기 전까지: `pending` 상태
   - 첫 청크를 받은 후: `success` 상태로 전환
   - 스트림이 끝날 때까지: `fetchStatus`는 `fetching` 유지

## 옵션 상세 설명

### 1. streamFn (필수)

```typescript
streamFn: (context: QueryFunctionContext) => Promise<AsyncIterable<TData>>
```

**역할**: 스트리밍할 데이터를 반환하는 AsyncIterable을 생성하는 함수
**받는 인자**: QueryFunctionContext (쿼리 키, signal 등 포함)

**쉽게 이해하기**: 
- AsyncIterable은 데이터를 한 번에 다 주지 않고, 준비되는 대로 조금씩 주는 방식
- 예: 채팅 메시지, 대용량 파일 다운로드, AI 응답 스트리밍 등

### 2. refetchMode (선택)

```typescript
refetchMode?: 'append' | 'reset' | 'replace'
```

**기본값**: `'reset'`

**모드별 동작**:
- **reset** (기본): 리페치 시 모든 데이터를 지우고 처음부터 다시 시작 (pending 상태로 돌아감)
- **append**: 리페치 시 새로운 데이터를 기존 데이터 뒤에 추가
- **replace**: 스트림이 완전히 끝난 후에만 모든 데이터를 한 번에 캐시에 저장

**사용 예시**:
```typescript
// 채팅 메시지를 계속 추가하는 경우
refetchMode: 'append'

// 완전히 새로운 데이터로 교체하는 경우
refetchMode: 'reset'

// 스트림이 끝날 때까지 기다렸다가 한 번에 업데이트
refetchMode: 'replace'
```

### 3. reducer (선택)

```typescript
reducer?: (accumulator: TData, chunk: TQueryFnData) => TData
```

**역할**: 스트리밍으로 받은 각 청크를 최종 데이터 형태로 변환

**기본 동작**: TData가 배열인 경우, 각 청크를 배열 끝에 추가
**필수 조건**: TData가 배열이 아닌 경우 반드시 제공해야 함

**쉽게 이해하기**:
```typescript
// 예시: 청크를 문자열로 합치기
reducer: (accumulated, newChunk) => {
  return accumulated + newChunk
}

// 예시: 청크를 객체에 병합
reducer: (accumulated, newChunk) => {
  return { ...accumulated, ...newChunk }
}
```

### 4. initialValue (선택)

```typescript
initialValue?: TData = TQueryFnData
```

**역할**: 첫 청크를 받기 전에 사용할 초기 데이터 정의
**기본값**: 빈 배열 `[]`
**필수 조건**: 커스텀 reducer를 사용하는 경우 반드시 제공해야 함

**사용 예시**:
```typescript
// 배열인 경우 (기본값 사용 가능)
initialValue: []

// 문자열로 합칠 경우
initialValue: ""

// 객체인 경우
initialValue: { messages: [], metadata: null }
```

## 실전 활용 예시

### AI 채팅 응답 스트리밍

```typescript
const chatQuery = queryOptions({
  queryKey: ['chat', messageId],
  queryFn: streamedQuery({
    streamFn: async () => fetchAIChatStream(messageId),
    refetchMode: 'reset', // 새 질문마다 이전 응답 지우기
  }),
})
```

### 무한 스크롤 데이터 추가

```typescript
const infiniteDataQuery = queryOptions({
  queryKey: ['items'],
  queryFn: streamedQuery({
    streamFn: async () => fetchItemsStream(),
    refetchMode: 'append', // 기존 데이터에 계속 추가
  }),
})
```

### 커스텀 데이터 형태

```typescript
const customQuery = queryOptions({
  queryKey: ['custom'],
  queryFn: streamedQuery({
    streamFn: async () => fetchStream(),
    reducer: (acc, chunk) => ({
      ...acc,
      count: acc.count + 1,
      items: [...acc.items, chunk],
    }),
    initialValue: { count: 0, items: [] },
  }),
})
```

## 주요 포인트 정리

1. **스트리밍이 필요한 경우**: 데이터가 점진적으로 생성되거나, 실시간 피드백이 중요한 경우 사용
2. **상태 관리**: 첫 데이터가 오면 바로 success로 처리되어 사용자에게 빠른 피드백 제공
3. **리페치 전략**: 데이터 추가/교체/초기화 방식을 refetchMode로 제어
4. **데이터 변환**: reducer로 청크를 원하는 형태로 가공 가능
5. **초기 상태**: initialValue로 로딩 중 표시할 기본값 설정
