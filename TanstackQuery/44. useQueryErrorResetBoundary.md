# useQueryErrorResetBoundary

## 📌 핵심 요약

`useQueryErrorResetBoundary`는 **가장 가까운 QueryErrorResetBoundary 내의 쿼리 에러를 초기화**하는 훅입니다. 만약 정의된 boundary가 없다면 전역으로 에러를 리셋합니다.

**주요 사용 목적**: React의 Error Boundary와 함께 사용하여 쿼리 에러 발생 시 "다시 시도" 기능을 구현

---

## 🔍 상세 설명

### 기본 개념

React Query는 쿼리 실행 중 에러가 발생하면 해당 에러 상태를 저장합니다. 이 에러 상태를 초기화하고 쿼리를 다시 실행하려면 에러를 리셋해야 합니다.

**어려운 부분 풀이:**
- **QueryErrorResetBoundary**: 쿼리 에러의 범위를 정의하는 컴포넌트입니다. 마치 React의 Error Boundary처럼 특정 영역의 에러만 관리할 수 있게 해줍니다.
- **가장 가까운(closest)**: 컴포넌트 트리에서 위쪽으로 올라가며 만나는 첫 번째 boundary를 의미합니다.
- **전역 리셋**: boundary가 없으면 애플리케이션 전체의 모든 쿼리 에러를 초기화합니다.

---

## 💡 사용 예시

```tsx
import { useQueryErrorResetBoundary } from '@tanstack/react-query'
import { ErrorBoundary } from 'react-error-boundary'

const App = () => {
  const { reset } = useQueryErrorResetBoundary()
  
  return (
    <ErrorBoundary
      onReset={reset}  // 에러 리셋 함수 연결
      fallbackRender={({ resetErrorBoundary }) => (
        <div>
          There was an error!
          <Button onClick={() => resetErrorBoundary()}>Try again</Button>
        </div>
      )}
    >
      <Page />
    </ErrorBoundary>
  )
}
```

### 코드 동작 방식

1. **`useQueryErrorResetBoundary()` 호출**
   - `reset` 함수를 반환받습니다.

2. **`ErrorBoundary`의 `onReset` prop에 연결**
   - React Error Boundary가 리셋될 때 쿼리 에러도 함께 초기화됩니다.

3. **사용자가 "Try again" 버튼 클릭**
   - `resetErrorBoundary()` 호출 → `onReset` 트리거 → `reset()` 실행
   - 쿼리 에러가 초기화되고 자동으로 재시도됩니다.

## 🎯 핵심 포인트

### 1. Error Boundary와의 결합
- React의 Error Boundary는 UI 에러를 잡습니다.
- `useQueryErrorResetBoundary`는 쿼리 에러를 초기화합니다.
- 두 개를 함께 사용하면 완전한 에러 복구 메커니즘 구현 가능

### 2. 범위 제어
```tsx
// 특정 영역만 리셋
<QueryErrorResetBoundary>
  <Component />  {/* 이 영역의 에러만 리셋 */}
</QueryErrorResetBoundary>

// 전역 리셋 (boundary 없이 사용)
const { reset } = useQueryErrorResetBoundary()  // 모든 쿼리 에러 리셋
```

### 3. 실무 활용
- **네트워크 에러 복구**: 일시적인 네트워크 문제 발생 시 재시도
- **인증 에러 처리**: 토큰 만료 후 로그인하고 다시 시도
- **부분 페이지 복구**: 전체 페이지 새로고침 없이 특정 영역만 재시도

## ⚠️ 주의사항

1. **`react-error-boundary` 라이브러리 필요**
   - 예시 코드의 `ErrorBoundary`는 별도 라이브러리입니다.
   - 직접 구현하거나 라이브러리 설치 필요

2. **자동 재시도와의 차이**
   - Query Retries: 자동으로 몇 번 재시도
   - Error Reset Boundary: **사용자 액션**으로 수동 재시도

3. **Suspense와의 차이**
   - Suspense: 로딩 상태 처리
   - Error Reset Boundary: 에러 상태 처리
