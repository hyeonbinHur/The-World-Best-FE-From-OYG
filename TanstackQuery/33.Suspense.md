# Suspense

## 개요

React Query는 React의 Suspense for Data Fetching API와 함께 사용할 수 있습니다. 이를 위한 전용 훅들이 제공됩니다.

## 제공되는 훅

- `useSuspenseQuery`
- `useSuspenseInfiniteQuery`
- `useSuspenseQueries`
- `useQuery().promise`와 `React.use()` (실험적 기능)

## 주요 특징

### 1. 상태 관리 변경

Suspense 모드를 사용하면:
- `status` 상태와 `error` 객체가 불필요
- 대신 `React.Suspense` 컴포넌트와 `fallback` prop 사용
- Error Boundary를 통해 에러 처리

### 2. 기본 사용법

```tsx
import { useSuspenseQuery } from '@tanstack/react-query'

const { data } = useSuspenseQuery({ queryKey, queryFn })
```

**TypeScript의 장점:**
- `data`가 항상 정의되어 있음 (undefined 체크 불필요)
- 에러와 로딩 상태는 Suspense와 Error Boundary가 처리

### 3. Suspense 모드의 제약사항

- 쿼리를 조건부로 활성화/비활성화 불가
- `placeholderData` 옵션 사용 불가
- UI가 fallback으로 대체되는 것을 방지하려면 `startTransition`으로 업데이트 감싸기

## throwOnError 기본값

기본적으로 모든 에러가 Error Boundary로 전달되지는 않습니다:

```plaintext
throwOnError: (error, query) => typeof query.state.data === 'undefined'
```

- 캐시에 데이터가 있으면 컴포넌트는 렌더링됨 (stale해도)
- 데이터가 없을 때만 에러를 throw

**모든 에러를 처리하려면:**

```tsx
const { data, error, isFetching } = useSuspenseQuery({ queryKey, queryFn })

if (error && !isFetching) {
  throw error
}
```

## Error Boundary 리셋

### QueryErrorResetBoundary 컴포넌트 사용

```tsx
import { QueryErrorResetBoundary } from '@tanstack/react-query'
import { ErrorBoundary } from 'react-error-boundary'

const App = () => (
  <QueryErrorResetBoundary>
    {({ reset }) => (
      <ErrorBoundary
        onReset={reset}
        fallbackRender={({ resetErrorBoundary }) => (
          <div>
            에러가 발생했습니다!
            <Button onClick={() => resetErrorBoundary()}>다시 시도</Button>
          </div>
        )}
      >
        <Page />
      </ErrorBoundary>
    )}
  </QueryErrorResetBoundary>
)
```

### useQueryErrorResetBoundary 훅 사용

```tsx
import { useQueryErrorResetBoundary } from '@tanstack/react-query'
import { ErrorBoundary } from 'react-error-boundary'

const App = () => {
  const { reset } = useQueryErrorResetBoundary()
  return (
    <ErrorBoundary
      onReset={reset}
      fallbackRender={({ resetErrorBoundary }) => (
        <div>
          에러가 발생했습니다!
          <Button onClick={() => resetErrorBoundary()}>다시 시도</Button>
        </div>
      )}
    >
      <Page />
    </ErrorBoundary>
  )
}
```

- 가장 가까운 `QueryErrorResetBoundary` 내의 쿼리 에러 리셋
- boundary가 없으면 전역적으로 리셋

## Fetch-on-render vs Render-as-you-fetch

**Fetch-on-render (기본 동작):**
- 컴포넌트 마운트 시 쿼리 실행 및 suspend

**Render-as-you-fetch (권장):**
- 라우팅 콜백이나 사용자 인터랙션 이벤트에서 Prefetching 구현
- 컴포넌트 마운트 전에 쿼리 로딩 시작

## 서버에서 Suspense 사용 (스트리밍)

NextJS에서 서버 사이드 Suspense 사용:

### 설정

```tsx
// app/providers.tsx
'use client'

import {
  isServer,
  QueryClient,
  QueryClientProvider,
} from '@tanstack/react-query'
import { ReactQueryStreamedHydration } from '@tanstack/react-query-next-experimental'

function makeQueryClient() {
  return new QueryClient({
    defaultOptions: {
      queries: {
        // SSR에서는 클라이언트에서 즉시 재fetch를 피하기 위해
        // staleTime을 0보다 크게 설정
        staleTime: 60 * 1000,
      },
    },
  })
}

let browserQueryClient: QueryClient | undefined = undefined

function getQueryClient() {
  if (isServer) {
    // 서버: 항상 새로운 query client 생성
    return makeQueryClient()
  } else {
    // 브라우저: query client가 없으면 생성
    // React가 초기 렌더링 중 suspend될 때 client를 다시 만들지 않도록 중요
    if (!browserQueryClient) browserQueryClient = makeQueryClient()
    return browserQueryClient
  }
}

export function Providers(props: { children: React.ReactNode }) {
  const queryClient = getQueryClient()

  return (
    <QueryClientProvider client={queryClient}>
      <ReactQueryStreamedHydration>
        {props.children}
      </ReactQueryStreamedHydration>
    </QueryClientProvider>
  )
}
```

## useQuery().promise와 React.use() (실험적)

### 활성화 방법

```tsx
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      experimental_prefetchInRender: true,
    },
  },
})
```

### 사용 예시

```tsx
import React from 'react'
import { useQuery } from '@tanstack/react-query'

function TodoList({ query }: { query: UseQueryResult<Todo[]> }) {
  const data = React.use(query.promise)

  return (
    <ul>
      {data.map((todo) => (
        <li key={todo.id}>{todo.title}</li>
      ))}
    </ul>
  )
}

export function App() {
  const query = useQuery({ queryKey: ['todos'], queryFn: fetchTodos })

  return (
    <>
      <h1>Todos</h1>
      <React.Suspense fallback={<div>Loading...</div>}>
        <TodoList query={query} />
      </React.Suspense>
    </>
  )
}
```

## 핵심 요점

1. **타입 안정성**: Suspense 훅은 `data`가 항상 정의되어 있음을 보장
2. **선택적 에러 처리**: 캐시에 데이터가 있으면 stale해도 렌더링
3. **에러 리셋**: `QueryErrorResetBoundary`로 재시도 기능 구현
4. **성능 최적화**: Prefetching으로 Render-as-you-fetch 패턴 구현
5. **SSR 지원**: NextJS에서 스트리밍 hydration 지원
6. **실험적 기능**: `React.use()`를 통한 promise 사용 가능
