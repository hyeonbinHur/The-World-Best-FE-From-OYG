# MutationCache

## 개요
MutationCache는 mutation들을 저장하는 저장소입니다. 일반적으로 MutationCache와 직접 상호작용하기보다는 QueryClient를 사용합니다.

## 기본 사용법
```tsx
import { MutationCache } from '@tanstack/react-query'

const mutationCache = new MutationCache({
  onError: (error) => {
    console.log(error)
  },
  onSuccess: (data) => {
    console.log(data)
  },
})
```

## 주요 메서드
- **getAll**: 캐시 내 모든 mutation 조회
- **subscribe**: mutation 캐시 변경사항 구독
- **clear**: 캐시 전체 초기화

## 옵션 설명

### onError (선택사항)
- **역할**: 어떤 mutation에서든 에러가 발생하면 호출
- **매개변수**: error, variables, onMutateResult, mutation, mutationFnContext
- **특징**: Promise를 반환하면 대기(await)됨

### onSuccess (선택사항)
- **역할**: 어떤 mutation이든 성공하면 호출
- **매개변수**: data, variables, onMutateResult, mutation, mutationFnContext
- **특징**: Promise를 반환하면 대기(await)됨

### onSettled (선택사항)
- **역할**: mutation이 완료되면 호출 (성공/실패 여부 무관)
- **매개변수**: data, error, variables, onMutateResult, mutation, mutationFnContext
- **특징**: Promise를 반환하면 대기(await)됨

### onMutate (선택사항)
- **역할**: mutation 실행 전에 호출
- **매개변수**: variables, mutation, mutationFnContext
- **특징**: Promise를 반환하면 대기(await)됨
- **제한**: 결과 값을 반환할 수 없음 (글로벌 콜백에서는)

## 글로벌 콜백의 특징

MutationCache의 콜백은 QueryClient의 defaultOptions와 다른 점:
1. **항상 호출됨**: defaultOptions는 각 Mutation에서 오버라이드 가능하지만, 글로벌 콜백은 항상 실행됩니다
2. **onMutate 제한**: 글로벌 onMutate는 결과를 반환할 수 없습니다

## 메서드 상세

### mutationCache.getAll()
캐시 내 모든 mutation을 반환합니다.

```tsx
const mutations = mutationCache.getAll()
```
- **반환값**: `Mutation[]` - 캐시의 모든 Mutation 인스턴스
- **사용 시기**: 일반적으로 자주 쓰이지 않지만, 드물게 mutation에 대한 추가 정보가 필요할 때 유용

### mutationCache.subscribe()
mutation 캐시 전체를 구독하여 mutation의 상태 변경, 업데이트, 추가, 제거와 같은 안전한 업데이트를 감지합니다.

```tsx
const callback = (event) => {
  console.log(event.type, event.mutation)
}

const unsubscribe = mutationCache.subscribe(callback)
```
- **매개변수**: `callback: (mutation?: MutationCacheNotifyEvent) => void`
- **반환값**: `unsubscribe: Function` - 구독 취소 함수

### mutationCache.clear()
캐시를 완전히 비우고 새로 시작합니다.

```tsx
mutationCache.clear()
```

## 핵심 요약

### 1. MutationCache란?
- Mutation들을 관리하는 중앙 저장소
- 보통 직접 사용하지 않고 QueryClient를 통해 사용

### 2. 글로벌 이벤트 핸들러
모든 mutation에 대해 공통으로 적용되는 4가지 생명주기 콜백:
- **onMutate**: 실행 전
- **onSuccess**: 성공 시
- **onError**: 실패 시
- **onSettled**: 완료 시 (성공/실패 무관)

### 3. 개별 Mutation vs 글로벌 콜백
- **개별 Mutation 옵션**: 오버라이드 가능, onMutate가 결과 반환 가능
- **글로벌 콜백**: 항상 실행됨, onMutate는 결과 반환 불가

### 4. 주요 활용 사례
- **에러 로깅**: 모든 mutation 에러를 한 곳에서 처리
- **성공 알림**: 모든 mutation 성공 시 토스트 메시지
- **디버깅**: subscribe로 실시간 mutation 변경사항 추적

### 어려운 개념 풀이

#### MutationFunctionContext란?
Mutation 함수의 실행 컨텍스트 정보를 담고 있는 객체입니다. Mutation이 실행되는 환경에 대한 메타데이터를 제공합니다.

#### onMutate vs onMutateResult
- **onMutate**: mutation 실행 전 호출되는 함수 (준비 단계)
- **onMutateResult**: onMutate 함수가 반환한 값 (낙관적 업데이트에 사용)

#### Promise를 반환하면 대기(await)됨
콜백 함수에서 비동기 작업을 수행하고 싶을 때, Promise를 반환하면 TanStack Query가 해당 작업이 완료될 때까지 기다립니다. 예를 들어 에러 발생 시 서버에 로그를 전송하는 비동기 작업을 완료한 후에 다음 단계로 진행하고 싶을 때 유용합니다.

#### subscribe의 활용
실시간으로 mutation 변경사항을 추적하고 싶을 때 사용합니다. 예를 들어 개발자 도구를 만들거나, mutation 상태 변화를 모니터링하는 대시보드를 구현할 때 유용합니다.
