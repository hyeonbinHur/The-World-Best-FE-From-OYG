# FocusManager (포커스 매니저)

## 핵심 요약

**FocusManager는 TanStack Query에서 애플리케이션의 포커스 상태를 관리하는 유틸리티입니다.**

- 기본 이벤트 리스너를 변경하거나 포커스 상태를 수동으로 제어할 수 있음
- 윈도우가 포커스를 받았는지 감지하여, 포커스 시 자동으로 쿼리를 다시 가져오는 기능 제어
- 4가지 주요 메서드: `setEventListener`, `subscribe`, `setFocused`, `isFocused`

---

## 주요 개념 설명

### 1. 포커스 상태란?
- 사용자가 현재 브라우저 탭/윈도우를 보고 있는지 여부
- 다른 탭으로 갔다가 돌아오면 포커스가 다시 활성화됨
- TanStack Query는 기본적으로 포커스 시 데이터를 자동으로 새로고침함

---

## 4가지 메서드 상세 설명

### 1. `setEventListener` - 커스텀 이벤트 리스너 설정

**용도**: 기본 포커스 감지 방식을 변경하고 싶을 때

**예제**:
```tsx
import { focusManager } from '@tanstack/react-query'

focusManager.setEventListener((handleFocus) => {
  // visibilitychange 이벤트로 포커스 감지
  if (typeof window !== 'undefined' && window.addEventListener) {
    window.addEventListener('visibilitychange', handleFocus, false)
  }

  return () => {
    // 새 핸들러가 설정되면 기존 것은 제거해야 함
    window.removeEventListener('visibilitychange', handleFocus)
  }
})
```

**쉽게 설명**:
- 브라우저가 포커스를 감지하는 방법을 직접 설정
- `visibilitychange` 이벤트: 사용자가 탭을 전환할 때 발생
- 반환하는 함수는 정리(cleanup) 함수 - 메모리 누수 방지

---

### 2. `subscribe` - 포커스 상태 변화 구독

**용도**: 포커스가 바뀔 때마다 특정 동작을 실행하고 싶을 때

**예제**:
```tsx
import { focusManager } from '@tanstack/react-query'

const unsubscribe = focusManager.subscribe((isVisible) => {
  console.log('현재 보이는 상태:', isVisible)
})

// 나중에 구독 해제
unsubscribe()
```

**쉽게 설명**:
- 포커스 상태가 변할 때마다 콜백 함수가 실행됨
- `isVisible`이 `true`면 사용자가 화면을 보고 있는 것
- `unsubscribe()`로 더 이상 감지하지 않도록 해제 가능

---

### 3. `setFocused` - 포커스 상태 수동 설정

**용도**: 자동 감지를 무시하고 강제로 포커스 상태를 설정할 때

**예제**:
```tsx
import { focusManager } from '@tanstack/react-query'

// 포커스 상태로 설정 (강제로 활성화)
focusManager.setFocused(true)

// 비포커스 상태로 설정 (강제로 비활성화)
focusManager.setFocused(false)

// 기본 포커스 감지 방식으로 되돌리기
focusManager.setFocused(undefined)
```

**쉽게 설명**:
- 테스트할 때 유용: 실제로 탭을 전환하지 않고도 포커스 동작 확인 가능
- `true`: "지금 사용자가 보고 있다"고 강제 설정
- `false`: "사용자가 안 보고 있다"고 강제 설정
- `undefined`: 다시 자동 감지 모드로 전환

---

### 4. `isFocused` - 현재 포커스 상태 확인

**용도**: 지금 현재 포커스 상태가 어떤지 확인할 때

**예제**:
```tsx
const isFocused = focusManager.isFocused()

if (isFocused) {
  console.log('사용자가 현재 화면을 보고 있습니다')
}
```

**쉽게 설명**:
- 단순히 현재 상태를 읽기만 함
- `true` 또는 `false` 반환
- 조건부 로직에 유용 (포커스 상태에 따라 다른 동작 실행)

---

## 실전 활용 시나리오

### 시나리오 1: 모바일 앱에서 앱이 백그라운드로 가는 경우 감지
```tsx
// 앱이 백그라운드로 가면 자동으로 포커스 false 처리
focusManager.setEventListener((handleFocus) => {
  document.addEventListener('visibilitychange', handleFocus)
  return () => document.removeEventListener('visibilitychange', handleFocus)
})
```

### 시나리오 2: 개발 중 자동 새로고침 비활성화
```tsx
// 개발할 때 탭 전환 시 자동 새로고침 막기
if (process.env.NODE_ENV === 'development') {
  focusManager.setFocused(false)
}
```

### 시나리오 3: 특정 조건에서만 포커스 동작 활성화
```tsx
focusManager.subscribe((isFocused) => {
  if (isFocused && userIsLoggedIn) {
    // 로그인한 사용자가 돌아왔을 때만 데이터 새로고침
    queryClient.invalidateQueries()
  }
})
```

## 핵심 정리

| 메서드 | 역할 | 언제 사용? |
|--------|------|-----------|
| `setEventListener` | 포커스 감지 방식 변경 | 기본 감지 방법을 커스터마이징할 때 |
| `subscribe` | 포커스 변화 감지 | 포커스 변화에 반응하는 로직이 필요할 때 |
| `setFocused` | 강제로 포커스 상태 설정 | 테스트 또는 특수한 상황에서 수동 제어가 필요할 때 |
| `isFocused` | 현재 포커스 상태 확인 | 조건부 로직에서 현재 상태를 판단할 때 |

---

## 주의사항

1. **이벤트 리스너는 반드시 정리해야 함**: `setEventListener`에서 반환하는 cleanup 함수를 제대로 작성하지 않으면 메모리 누수 발생 가능

2. **수동 설정 후 초기화**: `setFocused(true/false)`로 강제 설정했다면, 나중에 `setFocused(undefined)`로 되돌려야 정상 동작

3. **Window Focus Refetching과 연동**: FocusManager는 TanStack Query의 "Window Focus Refetching" 기능과 연동되어 작동함
