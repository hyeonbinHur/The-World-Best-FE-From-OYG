# 26. 필터 (Filters)

## 핵심 개념

TanStack Query의 일부 메서드는 `QueryFilters` 또는 `MutationFilters` 객체를 받아서 특정 조건에 맞는 쿼리나 뮤테이션을 선택적으로 처리할 수 있다.

---

## Query Filters (쿼리 필터)

쿼리 필터는 특정 조건에 맞는 쿼리를 찾기 위한 객체이다.

### 기본 사용 예시

```tsx
// 모든 쿼리 취소
await queryClient.cancelQueries()

// 'posts'로 시작하는 키를 가진 모든 비활성(inactive) 쿼리 제거
queryClient.removeQueries({ queryKey: ['posts'], type: 'inactive' })

// 모든 활성(active) 쿼리 refetch
await queryClient.refetchQueries({ type: 'active' })

// 'posts'로 시작하는 키를 가진 모든 활성 쿼리 refetch
await queryClient.refetchQueries({ queryKey: ['posts'], type: 'active' })
```

### Query Filter 속성

| 속성 | 타입 | 설명 |
|------|------|------|
| `queryKey` | `QueryKey` (optional) | 매칭할 쿼리 키를 정의 |
| `exact` | `boolean` (optional) | `true`로 설정하면 정확히 일치하는 쿼리 키만 반환 (포괄적 검색 X) |
| `type` | `'active' \| 'inactive' \| 'all'` | 기본값: `'all'`<br/>- `active`: 활성 쿼리만 매칭<br/>- `inactive`: 비활성 쿼리만 매칭 |
| `stale` | `boolean` (optional) | `true`: stale 쿼리만 매칭<br/>`false`: fresh 쿼리만 매칭 |
| `fetchStatus` | `FetchStatus` (optional) | - `fetching`: 현재 fetch 중인 쿼리<br/>- `paused`: fetch하려 했지만 일시 중지된 쿼리<br/>- `idle`: fetch하지 않는 쿼리 |
| `predicate` | `(query: Query) => boolean` | 모든 매칭 쿼리에 대한 최종 필터 함수. 다른 필터가 없으면 캐시의 모든 쿼리에 대해 실행됨 |

---

## Mutation Filters (뮤테이션 필터)

뮤테이션 필터는 특정 조건에 맞는 뮤테이션을 찾기 위한 객체이다.

### 기본 사용 예시

```tsx
// 모든 fetching 중인 뮤테이션 개수 가져오기
await queryClient.isMutating()

// mutationKey로 뮤테이션 필터링
await queryClient.isMutating({ mutationKey: ['post'] })

// predicate 함수를 사용한 뮤테이션 필터링
await queryClient.isMutating({
  predicate: (mutation) => mutation.state.variables?.id === 1,
})
```

### Mutation Filter 속성

| 속성 | 타입 | 설명 |
|------|------|------|
| `mutationKey` | `MutationKey` (optional) | 매칭할 뮤테이션 키를 정의 |
| `exact` | `boolean` (optional) | `true`로 설정하면 정확히 일치하는 뮤테이션 키만 반환 (포괄적 검색 X) |
| `status` | `MutationStatus` (optional) | 뮤테이션의 상태로 필터링 |
| `predicate` | `(mutation: Mutation) => boolean` | 모든 매칭 뮤테이션에 대한 최종 필터 함수. 다른 필터가 없으면 캐시의 모든 뮤테이션에 대해 실행됨 |

---

## Utils (유틸리티 함수)

### matchQuery

쿼리가 제공된 쿼리 필터와 일치하는지 여부를 나타내는 boolean을 반환한다.

```tsx
const isMatching = matchQuery(filters, query)
```

### matchMutation

뮤테이션이 제공된 뮤테이션 필터와 일치하는지 여부를 나타내는 boolean을 반환한다.

```tsx
const isMatching = matchMutation(filters, mutation)
```

## 요점 정리

| 항목 | 설명 |
|------|------|
| 목적 | 특정 조건에 맞는 쿼리/뮤테이션만 선택적으로 처리 (취소, 제거, refetch 등) |
| `queryKey` | 특정 키 패턴으로 쿼리 필터링 (기본적으로 포괄적 검색) |
| `exact: true` | 쿼리/뮤테이션 키가 정확히 일치하는 것만 선택 |
| `type` | 쿼리의 활성 상태로 필터링 (`active`, `inactive`, `all`) |
| `stale` | 쿼리가 stale인지 fresh인지로 필터링 |
| `fetchStatus` | 현재 fetch 상태로 필터링 (`fetching`, `paused`, `idle`) |
| `status` | 뮤테이션의 상태로 필터링 |
| `predicate` | 커스텀 조건 함수로 세밀한 필터링 가능 |
| Utils | `matchQuery`, `matchMutation`으로 필터 매칭 여부 확인 가능 |
| 활용 예시 | 특정 페이지의 쿼리만 무효화, 백그라운드 refetch 제어, 진행 중인 요청 취소 등 |
