# 14. 페이지네이션 쿼리 (Paginated / Lagged Queries)

## 기본 페이지네이션

페이지네이션은 매우 흔한 UI 패턴이며, TanStack Query에서는 쿼리 키에 페이지 정보를 포함하는 것만으로 동작한다.

```tsx
const result = useQuery({
  queryKey: ['projects', page],
  queryFn: () => fetchProjects(page),
})
```

### 문제점

위 방식의 문제는 **페이지가 바뀔 때마다 새로운 쿼리로 취급**되어, UI가 `success` → `pending` → `success` 상태를 반복하며 깜빡인다는 것이다. 사용자 입장에서는 페이지를 넘길 때마다 로딩 화면이 보이는 불편한 경험이 된다.

---

## placeholderData를 활용한 개선

`placeholderData` 옵션에 `keepPreviousData`를 설정하면 이 문제를 해결할 수 있다.

### keepPreviousData가 제공하는 것

| 기능 | 설명 |
|------|------|
| 이전 데이터 유지 | 새 데이터를 요청하는 동안 마지막으로 성공한 데이터를 계속 보여줌 |
| 자연스러운 전환 | 새 데이터가 도착하면 이전 데이터와 매끄럽게 교체됨 |
| `isPlaceholderData` 플래그 | 현재 보여지는 데이터가 placeholder(이전 데이터)인지 실제 새 데이터인지 구분 가능 |

### 전체 예제

```tsx
import { keepPreviousData, useQuery } from '@tanstack/react-query'
import React from 'react'

function Todos() {
  const [page, setPage] = React.useState(0)

  const fetchProjects = (page = 0) =>
    fetch('/api/projects?page=' + page).then((res) => res.json())

  const { isPending, isError, error, data, isFetching, isPlaceholderData } =
    useQuery({
      queryKey: ['projects', page],
      queryFn: () => fetchProjects(page),
      placeholderData: keepPreviousData,
    })

  return (
    <div>
      {isPending ? (
        <div>Loading...</div>
      ) : isError ? (
        <div>Error: {error.message}</div>
      ) : (
        <div>
          {data.projects.map((project) => (
            <p key={project.id}>{project.name}</p>
          ))}
        </div>
      )}
      <span>Current Page: {page + 1}</span>
      <button
        onClick={() => setPage((old) => Math.max(old - 1, 0))}
        disabled={page === 0}
      >
        Previous Page
      </button>
      <button
        onClick={() => {
          if (!isPlaceholderData && data.hasMore) {
            setPage((old) => old + 1)
          }
        }}
        disabled={isPlaceholderData || !data?.hasMore}
      >
        Next Page
      </button>
      {isFetching ? <span> Loading...</span> : null}
    </div>
  )
}
```

### 핵심 포인트

- `placeholderData: keepPreviousData`를 설정하면 페이지 전환 시 이전 데이터를 유지하므로 **깜빡임 없는 부드러운 UX**를 제공한다.
- `isPlaceholderData`가 `true`인 동안에는 Next Page 버튼을 비활성화하여, 아직 로딩 중인 상태에서 추가 페이지 이동을 방지한다.
- `isFetching`을 활용해 백그라운드에서 새 데이터를 가져오고 있음을 작은 로딩 인디케이터로 표시할 수 있다.

---

## useInfiniteQuery에서의 placeholderData

`placeholderData` 옵션은 `useInfiniteQuery` 훅에서도 동일하게 동작한다. infinite query의 키가 시간에 따라 변경되더라도, 사용자에게 캐시된 데이터를 계속 보여주면서 자연스러운 전환이 가능하다.

---

> **정리**: 페이지네이션에서 `keepPreviousData`는 페이지 전환 시 로딩 깜빡임을 없애는 핵심 옵션이다. `isPlaceholderData`로 현재 데이터 상태를 구분하고, `isFetching`으로 백그라운드 로딩을 표시하면 최적의 페이지네이션 UX를 구현할 수 있다.
