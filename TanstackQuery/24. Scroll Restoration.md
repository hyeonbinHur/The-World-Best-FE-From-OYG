# Scroll Restoration (스크롤 복원)

## 개요

전통적으로 웹 브라우저에서 이전에 방문한 페이지로 다시 이동하면, 해당 페이지를 떠나기 전의 정확한 위치로 스크롤이 되어 있는 것을 볼 수 있습니다. 이것을 **스크롤 복원(scroll restoration)**이라고 하며, 웹 애플리케이션이 클라이언트 사이드 데이터 페칭으로 이동하기 시작한 이후로 약간의 퇴보가 있었습니다. 하지만 TanStack Query를 사용하면 더 이상 그렇지 않습니다.

## TanStack Query의 스크롤 복원

TanStack Query는 스크롤 복원을 자체적으로 구현하지는 않지만, **SPA에서 스크롤 복원이 깨지는 가장 큰 원인 중 하나**를 제거합니다:

### 제거되는 문제: 리페치로 인한 UI 리셋

이전에 가져온 데이터를 캐시에 보관하고 (선택적으로 `placeholderData`를 사용하여), 페이지로 돌아가는 네비게이션이 안정적인 레이아웃으로 즉시 렌더링될 수 있도록 합니다. 이를 통해 라우터에서 처리하는 스크롤 복원을 신뢰할 수 있게 만듭니다.

### 지원하는 라우터

- React Router의 `ScrollRestoration`
- TanStack Router의 scroll restoration
- 작은 커스텀 히스토리 기반 솔루션

## 기본 동작

기본적으로 **모든 쿼리**(페이지네이션된 쿼리와 무한 쿼리 포함)에 대한 스크롤 복원이 TanStack Query에서 **그냥 작동합니다™️**.

### 작동 원리

쿼리 결과가 캐시되고 쿼리가 렌더링될 때 **동기적으로 검색**될 수 있기 때문입니다.

### 조건

쿼리가 충분히 오랫동안 캐시되고 (기본 시간은 **5분**) 가비지 컬렉션되지 않는 한, 스크롤 복원은 항상 기본적으로 작동합니다.

## 핵심 요점

1. **별도 구현 불필요**: TanStack Query는 스크롤 복원을 직접 구현하지 않음
2. **문제 해결**: 리페치로 인한 UI 리셋 문제를 제거
3. **캐싱의 힘**: 데이터 캐싱으로 안정적인 레이아웃 유지
4. **라우터 호환**: 주요 라우터들의 스크롤 복원 기능과 원활히 작동
5. **자동 지원**: 기본 설정으로도 모든 쿼리 타입에서 작동
