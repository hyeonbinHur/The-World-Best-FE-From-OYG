# 32. 기본 쿼리 함수 (Default Query Function)

## 핵심 개념

애플리케이션 전체에서 **동일한 쿼리 함수를 공유**하고, queryKey만으로 어떤 데이터를 가져올지 구분하고 싶을 때 사용한다. TanStack Query에 기본 쿼리 함수를 설정하면, 모든 쿼리에서 개별적으로 queryFn을 지정하지 않아도 된다.

---

## 기본 사용법

`QueryClient` 생성 시 `defaultOptions.queries.queryFn`에 기본 쿼리 함수를 정의한다. 이 함수는 자동으로 queryKey를 인자로 받는다.

```tsx
// queryKey를 받는 기본 쿼리 함수 정의
const defaultQueryFn = async ({ queryKey }) => {
  const { data } = await axios.get(
    `https://jsonplaceholder.typicode.com${queryKey[0]}`,
  )
  return data
}

// defaultOptions를 통해 앱 전체에 기본 쿼리 함수 제공
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      queryFn: defaultQueryFn,
    },
  },
})

function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <YourApp />
    </QueryClientProvider>
  )
}
```

---

## 사용 예시

기본 쿼리 함수가 설정되면, `useQuery`에서 queryKey만 전달해도 자동으로 해당 함수가 호출된다.

```tsx
// queryFn 없이 queryKey만 전달
function Posts() {
  const { status, data, error, isFetching } = useQuery({ 
    queryKey: ['/posts'] 
  })

  // ...
}

// queryFn을 생략하고 다른 옵션만 사용 가능
function Post({ postId }) {
  const { status, data, error, isFetching } = useQuery({
    queryKey: [`/posts/${postId}`],
    enabled: !!postId,  // queryFn 없이 다른 옵션만 지정
  })

  // ...
}
```

---

## 기본 쿼리 함수 오버라이드

필요한 경우 개별 쿼리에서 기본 쿼리 함수를 재정의할 수 있다. 일반적인 방식으로 queryFn을 제공하면 기본 함수 대신 해당 함수가 사용된다.

```tsx
const { data } = useQuery({
  queryKey: ['/posts'],
  queryFn: customQueryFunction,  // 기본 함수 대신 이 함수가 사용됨
})
```

## 요점 정리

| 항목 | 설명 |
|------|------|
| 목적 | 앱 전체에서 동일한 쿼리 함수를 공유하여 코드 중복 제거 |
| 설정 위치 | `QueryClient` 생성 시 `defaultOptions.queries.queryFn` |
| 자동 전달 인자 | `queryKey`를 포함한 QueryFunctionContext가 자동으로 전달됨 |
| 사용 방식 | `useQuery`에서 queryFn을 생략하고 queryKey만 전달 |
| 오버라이드 | 개별 쿼리에서 queryFn을 명시하면 기본 함수 대신 해당 함수가 사용됨 |
| 주요 장점 | queryKey 기반으로 API 엔드포인트를 구성하는 RESTful API 패턴에 적합 |
| 적용 범위 | QueryClientProvider 내부의 모든 쿼리에 적용 |

---

## 활용 팁

- RESTful API처럼 URL 패턴이 일관적인 경우 매우 유용
- queryKey의 첫 번째 요소를 엔드포인트로 사용하는 패턴이 일반적
- GraphQL이나 복잡한 쿼리 로직이 필요한 경우 개별 queryFn 사용 권장
