# QueryErrorResetBoundary

## 번역

쿼리에서 `suspense` 또는 `throwOnError`를 사용할 때, 에러가 발생한 후 다시 렌더링할 때 쿼리에게 재시도하고 싶다는 것을 알려주는 방법이 필요합니다. `QueryErrorResetBoundary` 컴포넌트를 사용하면 컴포넌트 경계 내의 모든 쿼리 에러를 리셋할 수 있습니다.

### 예시 코드

```tsx
import { QueryErrorResetBoundary } from '@tanstack/react-query'
import { ErrorBoundary } from 'react-error-boundary'

const App = () => (
  <QueryErrorResetBoundary>
    {({ reset }) => (
      <ErrorBoundary
        onReset={reset}
        fallbackRender={({ resetErrorBoundary }) => (
          <div>
            There was an error!
            <Button onClick={() => resetErrorBoundary()}>Try again</Button>
          </div>
        )}
      >
        <Page />
      </ErrorBoundary>
    )}
  </QueryErrorResetBoundary>
)
```

---

## 핵심 요약

### 목적
- Suspense 또는 throwOnError 사용 시 에러 복구 메커니즘 제공
- 사용자가 에러 발생 후 쿼리를 다시 시도할 수 있게 함

### 주요 개념

**1. 왜 필요한가?**
- `suspense: true` 또는 `throwOnError: true` 옵션을 사용하면 쿼리가 에러를 throw함
- React ErrorBoundary가 에러를 잡지만, 쿼리는 여전히 에러 상태로 남아있음
- 단순히 컴포넌트를 다시 렌더링해도 쿼리가 재시도되지 않음
- **QueryErrorResetBoundary**가 쿼리의 에러 상태를 초기화해줌

**2. 동작 방식**
- `QueryErrorResetBoundary`가 `reset` 함수를 제공함
- 이 `reset` 함수를 `ErrorBoundary`의 `onReset`에 연결
- 사용자가 "다시 시도" 버튼을 클릭하면:
  1. ErrorBoundary가 리셋됨 (UI 복구)
  2. QueryErrorResetBoundary가 쿼리 에러 상태를 초기화
  3. 컴포넌트가 다시 렌더링되면서 쿼리가 재시도됨

**3. 사용 패턴**
```
QueryErrorResetBoundary (쿼리 에러 리셋 관리)
  └─ ErrorBoundary (UI 에러 처리)
      └─ 실제 컴포넌트 (Page)
```

### 쉽게 이해하기

**문제 상황:**
- 쿼리가 실패 → ErrorBoundary가 에러 UI 표시
- 사용자가 "다시 시도" 클릭 → ErrorBoundary만 리셋
- **하지만 쿼리는 여전히 에러 상태!** → 다시 렌더링해도 똑같은 에러 발생

**해결 방법:**
- QueryErrorResetBoundary가 쿼리의 에러 상태도 함께 리셋
- ErrorBoundary + QueryErrorResetBoundary 함께 사용
- 사용자가 버튼 클릭 → UI도 리셋, 쿼리도 리셋 → 새로운 시도 가능

### 실전 활용

**언제 사용?**
- Suspense 기반 데이터 페칭 사용 시
- throwOnError로 에러를 throw하는 쿼리 사용 시
- 사용자에게 재시도 옵션을 제공하고 싶을 때

**함께 사용하는 라이브러리:**
- `react-error-boundary` (가장 일반적)
- 또는 직접 만든 ErrorBoundary 클래스 컴포넌트

**주의사항:**
- QueryErrorResetBoundary는 **경계 내부**의 모든 쿼리 에러를 리셋함
- 여러 개의 QueryErrorResetBoundary를 중첩해서 사용 가능 (더 세밀한 제어)
