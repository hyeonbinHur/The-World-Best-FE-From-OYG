# 16. 초기 쿼리 데이터 (Initial Query Data)

## 초기 데이터를 제공하는 방법

캐시에 데이터가 필요하기 전에 미리 초기 데이터를 제공할 수 있다.

| 방식 | 설명 |
|------|------|
| **선언적 (Declarative)** | `initialData` 옵션을 사용해 캐시가 비어있을 때 초기 데이터를 설정 |
| **명령적 (Imperative)** | `queryClient.prefetchQuery`로 사전 fetch |
| **명령적 (Imperative)** | `queryClient.setQueryData`로 캐시에 수동 삽입 |

---

## initialData로 쿼리 초기 데이터 설정

앱에 이미 초기 데이터가 있는 경우, `initialData` 옵션을 사용하면 초기 로딩 상태를 건너뛸 수 있다.

> **주의**: `initialData`는 캐시에 영구 저장된다. 불완전하거나 임시 데이터는 `placeholderData`를 사용해야 한다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
})
```

---

## staleTime과 initialDataUpdatedAt

기본적으로 `initialData`는 **방금 fetch된 것처럼 완전히 fresh한 데이터**로 취급된다.

### 기본 동작 (staleTime: 0)

`staleTime`을 설정하지 않으면 마운트 즉시 refetch가 발생한다.

```tsx
// initialTodos를 즉시 보여주지만, 마운트 후 바로 refetch
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
})
```

### staleTime 설정 시

`staleTime: 1000`을 설정하면, 1000ms 동안은 fresh로 간주되어 refetch하지 않는다.

```tsx
// initialTodos를 즉시 보여주고, 1000ms 이후 상호작용 이벤트가 있어야 refetch
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
  staleTime: 1000,
})
```

### initialDataUpdatedAt 사용

`initialData`가 완전히 fresh하지 않은 경우, `initialDataUpdatedAt`으로 초기 데이터가 마지막으로 업데이트된 시점(JS 타임스탬프, 밀리초)을 전달할 수 있다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
  staleTime: 60 * 1000, // 1분
  initialDataUpdatedAt: initialTodosUpdatedTimestamp, // eg. 1608412420052
})
```

> `staleTime`은 "데이터가 얼마나 fresh해야 하는가"를 결정하고, `initialDataUpdatedAt`은 "초기 데이터가 언제 업데이트되었는가"를 알려준다. 쿼리가 이 두 값을 비교해서 마운트 시 refetch 여부를 스스로 판단한다.

## Initial Data Function (초기 데이터 함수)

초기 데이터를 가져오는 과정이 무거운 경우, 함수로 전달하면 **쿼리가 초기화될 때 한 번만** 실행된다. 매 렌더마다 실행되지 않으므로 메모리/CPU를 절약할 수 있다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: () => getExpensiveTodos(),
})
```

## 캐시에서 초기 데이터 가져오기

다른 쿼리의 캐시 결과를 초기 데이터로 사용할 수 있다. 예를 들어, todos 목록 캐시에서 특정 todo를 꺼내 개별 todo 쿼리의 초기 데이터로 사용:

```tsx
const result = useQuery({
  queryKey: ['todo', todoId],
  queryFn: () => fetch('/todos'),
  initialData: () => {
    return queryClient.getQueryData(['todos'])?.find((d) => d.id === todoId)
  },
})
```

### initialDataUpdatedAt과 함께 사용

캐시에서 가져온 초기 데이터는 이미 오래되었을 수 있다. 인위적인 `staleTime` 대신, 원본 쿼리의 `dataUpdatedAt`을 `initialDataUpdatedAt`에 전달하면 쿼리가 refetch 시점을 정확히 판단할 수 있다.

```tsx
const result = useQuery({
  queryKey: ['todos', todoId],
  queryFn: () => fetch(`/todos/${todoId}`),
  initialData: () =>
    queryClient.getQueryData(['todos'])?.find((d) => d.id === todoId),
  initialDataUpdatedAt: () =>
    queryClient.getQueryState(['todos'])?.dataUpdatedAt,
})
```

---

## 조건부 캐시 초기 데이터

원본 쿼리의 캐시가 너무 오래되었으면 초기 데이터를 사용하지 않고, 서버에서 새로 fetch하도록 할 수도 있다. `queryClient.getQueryState`로 `dataUpdatedAt`을 확인하여 판단:

```tsx
const result = useQuery({
  queryKey: ['todo', todoId],
  queryFn: () => fetch(`/todos/${todoId}`),
  initialData: () => {
    const state = queryClient.getQueryState(['todos'])

    // 쿼리가 존재하고, 데이터가 10초 이내인 경우에만 사용
    if (state && Date.now() - state.dataUpdatedAt <= 10 * 1000) {
      return state.data.find((d) => d.id === todoId)
    }

    // undefined 반환 → 하드 로딩 상태에서 fetch 시작
  },
})
```

## 정리

| 개념 | 설명 |
|------|------|
| `initialData` | 캐시에 저장되는 초기 데이터. 불완전한 데이터는 `placeholderData` 사용 |
| `staleTime` | 데이터가 fresh로 간주되는 시간 |
| `initialDataUpdatedAt` | 초기 데이터의 마지막 업데이트 시점 (JS 밀리초 타임스탬프) |
| Initial Data Function | 무거운 초기 데이터를 함수로 전달하여 한 번만 실행 |
| 캐시에서 가져오기 | `queryClient.getQueryData`로 다른 쿼리의 캐시를 초기 데이터로 활용 |
| 조건부 캐시 | `queryClient.getQueryState`의 `dataUpdatedAt`으로 캐시 신선도를 판단하여 사용 여부 결정 |
