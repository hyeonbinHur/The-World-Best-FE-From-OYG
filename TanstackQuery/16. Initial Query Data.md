# 16. 초기 쿼리 데이터 (Initial Query Data)

## 개요

쿼리 캐시에 초기 데이터를 미리 제공하는 방법은 크게 두 가지로 나뉜다:

| 방식 | 설명 |
|------|------|
| **선언적 (Declarative)** | `initialData` 옵션으로 캐시가 비어 있을 때 초기 데이터를 미리 채움 |
| **명령적 (Imperative)** | `queryClient.prefetchQuery`로 프리페치하거나, `queryClient.setQueryData`로 수동으로 캐시에 데이터를 넣음 |

> **주의**: `initialData`는 캐시에 영구 저장된다. 불완전한 데이터나 placeholder용 데이터는 `placeholderData` 옵션을 사용해야 한다.

---

## initialData로 쿼리 미리 채우기

앱에서 이미 초기 데이터를 갖고 있다면 `initialData` 옵션으로 직접 전달할 수 있다. 이렇게 하면 초기 로딩 상태를 건너뛸 수 있다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
})
```

---

## staleTime과 initialDataUpdatedAt

기본적으로 `initialData`는 **방금 fetch된 것처럼 완전히 fresh한 데이터**로 취급된다. 이것은 `staleTime` 옵션의 해석에도 영향을 준다.

### 케이스 1: staleTime 없음 (기본값 0)

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
})
```

- `initialTodos`를 즉시 표시하지만, **마운트 직후 바로 refetch**가 발생한다.

### 케이스 2: staleTime 설정

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
  staleTime: 1000,
})
```

- `initialTodos`를 즉시 표시하고, **1000ms 동안은 fresh로 간주**하여 refetch하지 않는다.

### 케이스 3: initialDataUpdatedAt 사용

`initialData`가 완전히 최신이 아닐 경우, `initialDataUpdatedAt` 옵션으로 데이터가 마지막으로 업데이트된 시점(JS 타임스탬프, ms 단위)을 전달할 수 있다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
  staleTime: 60 * 1000, // 1분
  initialDataUpdatedAt: initialTodosUpdatedTimestamp, // eg. 1608412420052
})
```

- `staleTime`은 본래 목적(데이터 신선도 기준)대로 사용하면서, `initialDataUpdatedAt`으로 초기 데이터의 실제 업데이트 시점을 알려준다.
- 쿼리가 스스로 refetch 필요 여부를 판단할 수 있게 된다.

> **팁**: 프리페치된 데이터로 취급하고 싶다면 `prefetchQuery` 또는 `fetchQuery` API를 사용하여 `staleTime`과 `initialData`를 독립적으로 설정하는 것이 좋다.

## Initial Data Function (초기 데이터 함수)

초기 데이터를 가져오는 과정이 비용이 크다면, **함수로 전달**하여 쿼리가 초기화될 때 한 번만 실행되도록 할 수 있다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: () => getExpensiveTodos(),
})
```

- 매 렌더마다 실행되지 않고, **쿼리 초기화 시 단 한 번**만 실행된다.

---

## 캐시에서 초기 데이터 가져오기

다른 쿼리의 캐시된 결과에서 초기 데이터를 가져올 수 있다. 예를 들어, todos 목록 캐시에서 개별 todo 항목을 꺼내서 사용할 수 있다.

```tsx
const result = useQuery({
  queryKey: ['todo', todoId],
  queryFn: () => fetch('/todos'),
  initialData: () => {
    return queryClient.getQueryData(['todos'])?.find((d) => d.id === todoId)
  },
})
```

---

## 캐시에서 가져올 때 initialDataUpdatedAt 함께 사용

캐시에서 초기 데이터를 가져오면 원본 쿼리가 오래된 데이터일 수 있다. 인위적인 `staleTime` 대신, 원본 쿼리의 `dataUpdatedAt`을 `initialDataUpdatedAt`에 전달하는 것이 좋다.

```tsx
const result = useQuery({
  queryKey: ['todos', todoId],
  queryFn: () => fetch(`/todos/${todoId}`),
  initialData: () =>
    queryClient.getQueryData(['todos'])?.find((d) => d.id === todoId),
  initialDataUpdatedAt: () =>
    queryClient.getQueryState(['todos'])?.dataUpdatedAt,
})
```

- 쿼리 인스턴스가 refetch 필요 여부를 판단하는 데 필요한 모든 정보를 제공한다.

---

## 조건부 캐시 초기 데이터 (Conditional Initial Data from Cache)

원본 캐시 데이터가 너무 오래됐다면, 아예 사용하지 않고 서버에서 직접 가져오도록 할 수 있다. `queryClient.getQueryState`를 사용하여 신선도를 직접 판단한다.

```tsx
const result = useQuery({
  queryKey: ['todo', todoId],
  queryFn: () => fetch(`/todos/${todoId}`),
  initialData: () => {
    const state = queryClient.getQueryState(['todos'])

    // 쿼리가 존재하고 데이터가 10초 이내라면 캐시 데이터 사용
    if (state && Date.now() - state.dataUpdatedAt <= 10 * 1000) {
      return state.data.find((d) => d.id === todoId)
    }

    // 그렇지 않으면 undefined 반환 → 하드 로딩 상태에서 fetch
  },
})
```

## 요약 정리

| 항목 | 설명 |
|------|------|
| `initialData` | 캐시에 저장되는 초기 데이터. 불완전한 데이터에는 부적합 |
| `placeholderData` | 캐시에 저장되지 않는 임시 표시용 데이터 |
| `staleTime` | 데이터를 fresh로 간주하는 시간 |
| `initialDataUpdatedAt` | 초기 데이터의 실제 업데이트 시점 (JS 타임스탬프) |
| `initialData` 함수 | 비용이 큰 연산을 쿼리 초기화 시 한 번만 실행 |
| 캐시에서 가져오기 | 다른 쿼리의 캐시 데이터를 초기 데이터로 활용 |
| 조건부 캐시 사용 | `getQueryState`로 신선도를 판단하여 사용 여부 결정 |
