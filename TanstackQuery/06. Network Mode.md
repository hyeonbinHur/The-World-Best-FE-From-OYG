Tanstack Query는 네트워크 연결이 없을 때, Query와 Mutation이 어떻게 동작해야 하는지를 구분하기 위해 세 가지 네트워크 모드를 제공합니다.

```ts
useQuery({
  queryKey: ["todos"],
  queryFn: fetchTodos,
  networkMode: "offlineFirst",
});
```

## Online (일시 중지)

-> 네트워크가 없으면 아예 실행하지 않으며, status는 기존을 유지하고 fetchStatus는 paused된다.

-> 네트워크 재연결 되면 retry를 하고 재연결 전까지는 retry를 일시 중지한다. 이는 refetchOnReconnect와는 독립적이며, 네트워크 재연결이후 실행은 re가 아닌 continue로 간주하기 때문이다 (만약 네트워크 재연결 전에 명시적 캔슬하면 retry 또한 발생하지 않는다.)

## Always (일시 중지 안함)

-> 일부 네트워크 요청이 필요 없는 promise가 있을 수 있기 때문에 network 연결과는 별개로 무조건 의도한대로 실행하고 실패하고 retry한다.
-> 네트워크 연결과는 별개이기 때문에 refetchOnReconnect는 해당 모드에서 false를 기본으로 한다.

## OfflineFirst (첫시도 후 일시 중지)

-> 일부 서비스는 service Worker나, http 캐시를 이용해서 데이터를 캐싱하고 이런 경우 네트워크 연결이 없더라도 성공할 수 있다. 때문에, 이러한 상황을 대비해서 OfflineFirst는 네트워크 연결이 없더라도 한번은 시도하고, 실패한다면 retry는 하지 않는다.

---

#### DevTools

DevTools에서는 네트워크 연결이 없어서 fetch가 보류중인 Query를 paused 상태로 표시한다.
