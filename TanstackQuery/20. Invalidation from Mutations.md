# 20. Mutation에서의 쿼리 무효화 (Invalidations from Mutations)

## 핵심 개념

쿼리를 무효화(invalidate)하는 것은 전투의 절반에 불과하다. **언제** 무효화할지 아는 것이 나머지 절반이다. 보통 앱에서 mutation이 성공하면, 관련된 쿼리들을 무효화하고 다시 fetch해서 mutation으로 인한 변경 사항을 반영해야 할 가능성이 매우 높다.

---

## 기본 사용법

예를 들어, 새로운 todo를 추가하는 mutation이 있다고 가정하자:

```tsx
const mutation = useMutation({ mutationFn: postTodo })
```

`postTodo` mutation이 성공하면, 모든 todos 쿼리를 무효화하고 다시 fetch해서 새로운 todo 항목을 보여줘야 한다. 이를 위해 `useMutation`의 `onSuccess` 옵션과 `queryClient`의 `invalidateQueries` 함수를 함께 사용한다:

```tsx
import { useMutation, useQueryClient } from '@tanstack/react-query'

const queryClient = useQueryClient()

// mutation 성공 시, 'todos' 또는 'reminders' 쿼리 키를 가진 쿼리를 무효화
const mutation = useMutation({
  mutationFn: addTodo,
  onSuccess: async () => {
    // 단일 쿼리 무효화
    await queryClient.invalidateQueries({ queryKey: ['todos'] })

    // 여러 쿼리를 동시에 무효화
    await Promise.all([
      queryClient.invalidateQueries({ queryKey: ['todos'] }),
      queryClient.invalidateQueries({ queryKey: ['reminders'] }),
    ])
  },
})
```

---

## 주요 포인트

| 포인트 | 설명 |
|--------|------|
| `onSuccess`에서 Promise 반환 | `onSuccess`에서 Promise를 반환하면, 데이터가 업데이트될 때까지 mutation이 완전히 완료되지 않는다 (`isPending`이 `onSuccess`가 fulfilled될 때까지 `true`로 유지) |
| 콜백 위치 | `useMutation` 훅에서 제공하는 모든 콜백(`onSuccess`, `onError`, `onSettled` 등)에서 무효화를 연결할 수 있다 |
| 자동 무효화 | mutation 후 자동으로 쿼리를 무효화하는 기법은 TkDodo의 "Automatic Query Invalidation after Mutations" 글을 참고 |

---

## 요약 정리

1. **Mutation 성공 후 관련 쿼리 무효화**가 일반적인 패턴이다
2. `useMutation`의 `onSuccess` 콜백에서 `queryClient.invalidateQueries()`를 호출하면 된다
3. 여러 쿼리를 동시에 무효화하려면 `Promise.all`을 사용한다
4. `onSuccess`에서 `await`를 사용하면 데이터 갱신이 완료될 때까지 mutation의 `isPending`이 `true`로 유지되어 UI 일관성을 보장할 수 있다
5. `onSuccess` 외에도 `onError`, `onSettled` 등 다양한 콜백에서 무효화를 트리거할 수 있다
