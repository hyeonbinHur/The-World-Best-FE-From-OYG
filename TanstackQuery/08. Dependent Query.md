### useQuery 종속 쿼리

종속 (또는 직렬)쿼리는 Parallel과 반대로, 이전 Query가 완료되어야한 직렬적으로 실행되는 쿼리를 말하며 enabled 옵션으로 실행 가능합니다.

```tsx
// 유저 정보를 가져온다
const { data: user } = useQuery({
  queryKey: ["user", email],
  queryFn: getUserByEmail,
});

const userId = user?.id;

// 그 다음 유저의 프로젝트를 가져온다
const {
  status,
  fetchStatus,
  data: projects,
} = useQuery({
  queryKey: ["projects", userId],
  queryFn: getProjectsByUser,
  // userId가 존재할 때까지 쿼리가 실행되지 않는다
  enabled: !!userId,
});
```

### useQueries의 종속 쿼리

```tsx
// 유저 ID 목록을 가져온다
const { data: userIds } = useQuery({
  queryKey: ["users"],
  queryFn: getUsersData,
  select: (users) => users.map((user) => user.id),
});

// 그 다음 각 유저의 메시지를 가져온다
const usersMessages = useQueries({
  queries: userIds
    ? userIds.map((id) => {
        return {
          queryKey: ["messages", id],
          queryFn: () => getMessagesByUsers(id),
        };
      })
    : [], // userIds가 undefined이면 빈 배열이 반환된다
});
```

### 성능 참고사항

한줄요약: 백엔드 api를 만들때 애초에 api를 다른 api 결과 의존성을 최소화 하는 방식으로 설계해야한다

이메일로 유저아이디 찾기 -> 유저 아이디로 프로젝트 찾기: 이런식이 아니라
유저이메일로 프로젝트 찾기: 이런식의 설계가 필요하다.
