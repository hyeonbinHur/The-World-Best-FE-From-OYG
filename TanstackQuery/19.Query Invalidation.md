# 19. 쿼리 무효화 (Query Invalidation)

## 쿼리 무효화란?

쿼리가 stale 상태가 될 때까지 기다렸다가 다시 fetch하는 것이 항상 적절한 것은 아니다. 사용자의 특정 행동으로 인해 쿼리 데이터가 이미 오래된 것이 확실할 때, `QueryClient`의 `invalidateQueries` 메서드를 사용하여 쿼리를 stale로 표시하고 자동으로 refetch할 수 있다.

```tsx
// 캐시의 모든 쿼리를 무효화
queryClient.invalidateQueries()

// 'todos'로 시작하는 키를 가진 모든 쿼리를 무효화
queryClient.invalidateQueries({ queryKey: ['todos'] })
```

> **참고**: 정규화된 캐시를 사용하는 다른 라이브러리들은 새 데이터로 로컬 쿼리를 직접 업데이트하려 하지만, TanStack Query는 **타겟 무효화 → 백그라운드 refetch → 원자적 업데이트** 방식을 권장한다.

---

## 무효화 시 동작

쿼리가 `invalidateQueries`로 무효화되면 두 가지 일이 발생한다:

| 동작 | 설명 |
|------|------|
| stale 표시 | `useQuery` 등에서 설정한 `staleTime` 설정을 무시하고 즉시 stale 상태로 변경 |
| 백그라운드 refetch | 해당 쿼리가 현재 `useQuery` 등으로 렌더링 중이면 백그라운드에서 자동 refetch |

## 쿼리 매칭 (Query Matching with invalidateQueries)

`invalidateQueries`, `removeQueries` 등의 API에서는 **접두사 매칭**, **정확한 매칭**, **predicate 함수** 등 다양한 방식으로 쿼리를 필터링할 수 있다.

### 1. 접두사 매칭 (기본)

queryKey 배열의 앞부분이 일치하는 모든 쿼리를 무효화한다.

```tsx
import { useQuery, useQueryClient } from '@tanstack/react-query'

const queryClient = useQueryClient()

queryClient.invalidateQueries({ queryKey: ['todos'] })

// 아래 두 쿼리 모두 무효화됨
const todoListQuery = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodoList,
})
const todoListQuery = useQuery({
  queryKey: ['todos', { page: 1 }],
  queryFn: fetchTodoList,
})
```

### 2. 더 구체적인 키로 매칭

특정 변수를 포함한 쿼리만 무효화할 수 있다.

```tsx
queryClient.invalidateQueries({
  queryKey: ['todos', { type: 'done' }],
})

// 무효화됨 ✅
const todoListQuery = useQuery({
  queryKey: ['todos', { type: 'done' }],
  queryFn: fetchTodoList,
})

// 무효화되지 않음 ❌
const todoListQuery = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodoList,
})
```

### 3. 정확한 매칭 (`exact: true`)

하위 키나 추가 변수가 없는 정확히 일치하는 쿼리만 무효화한다.

```tsx
queryClient.invalidateQueries({
  queryKey: ['todos'],
  exact: true,
})

// 무효화됨 ✅
const todoListQuery = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodoList,
})

// 무효화되지 않음 ❌
const todoListQuery = useQuery({
  queryKey: ['todos', { type: 'done' }],
  queryFn: fetchTodoList,
})
```

### 4. Predicate 함수로 매칭

더 세밀한 제어가 필요할 때, `predicate` 함수를 전달하여 각 `Query` 인스턴스를 직접 검사할 수 있다.

```tsx
queryClient.invalidateQueries({
  predicate: (query) =>
    query.queryKey[0] === 'todos' && query.queryKey[1]?.version >= 10,
})

// 무효화됨 ✅ (version: 20)
const todoListQuery = useQuery({
  queryKey: ['todos', { version: 20 }],
  queryFn: fetchTodoList,
})

// 무효화됨 ✅ (version: 10)
const todoListQuery = useQuery({
  queryKey: ['todos', { version: 10 }],
  queryFn: fetchTodoList,
})

// 무효화되지 않음 ❌ (version: 5)
const todoListQuery = useQuery({
  queryKey: ['todos', { version: 5 }],
  queryFn: fetchTodoList,
})
```

## 요점 정리

| 항목 | 설명 |
|------|------|
| **목적** | 데이터가 오래된 것이 확실할 때 수동으로 쿼리를 stale 처리하고 refetch를 트리거 |
| **기본 동작** | stale 표시 + 렌더링 중인 쿼리는 백그라운드 refetch |
| **접두사 매칭** | `queryKey: ['todos']`로 `['todos']`, `['todos', { page: 1 }]` 등 모두 매칭 |
| **구체적 매칭** | `queryKey: ['todos', { type: 'done' }]`로 정확한 변수 포함 쿼리만 매칭 |
| **정확한 매칭** | `exact: true` 옵션으로 하위 키 없이 정확히 일치하는 쿼리만 매칭 |
| **Predicate** | 함수를 전달하여 커스텀 조건으로 무효화 대상 결정 |
| **vs 정규화 캐시** | TanStack Query는 정규화 캐시 대신 타겟 무효화 + 백그라운드 refetch 방식을 사용 |
