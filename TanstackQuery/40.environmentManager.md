# environmentManager

## 핵심 개념

`environmentManager`는 TanStack Query가 현재 런타임 환경이 서버 사이드인지 클라이언트 사이드인지를 판단하는 방식을 관리하는 매니저입니다.

## 주요 기능

### 1. 서버 환경 감지

기본적으로 `query-core`에서 제공하는 `isServer` 유틸리티와 동일한 방식으로 서버를 감지합니다.

### 2. 커스텀 환경 감지 설정

일반적인 브라우저/서버 환경이 아닌 특수한 런타임(예: 브라우저 확장 프로그램 워커)에서 서버 감지 로직을 전역적으로 재정의할 수 있습니다.

## API 메서드

### `environmentManager.isServer()`

현재 런타임이 서버 환경으로 간주되는지 여부를 반환합니다.

**사용 예시:**
```tsx
import { environmentManager } from '@tanstack/react-query'

const server = environmentManager.isServer()
```

### `environmentManager.setIsServer()`

서버 체크 로직을 전역적으로 재정의합니다.

**옵션:**
- `isServerValue: () => boolean` - 서버 여부를 판단하는 함수

**사용 예시 1: 커스텀 서버 감지 로직**
```tsx
import { environmentManager } from '@tanstack/react-query'

// Chrome 확장 프로그램 환경을 고려한 커스텀 로직
environmentManager.setIsServer(() => {
  return typeof window === 'undefined' && !('chrome' in globalThis)
})
```

**사용 예시 2: 기본 동작 복원**
```tsx
import { environmentManager, isServer } from '@tanstack/react-query'

// 기본 동작으로 되돌리기
environmentManager.setIsServer(() => isServer)
```

## 언제 사용하나요?

### 일반적인 웹 애플리케이션
대부분의 경우 기본 설정을 그대로 사용하면 됩니다. TanStack Query가 자동으로 브라우저와 서버 환경을 구분합니다.

### 특수 환경
다음과 같은 특수한 런타임 환경에서 사용합니다:
- **브라우저 확장 프로그램**: Service Worker나 Background Script처럼 `window` 객체가 없지만 서버가 아닌 환경
- **Edge Runtime**: Cloudflare Workers, Vercel Edge Functions 등
- **임베디드 JavaScript 환경**: Electron, React Native 등

## 핵심 요약

1. **목적**: 현재 코드가 서버에서 실행되는지, 클라이언트에서 실행되는지 판단
2. **기본 동작**: `window` 객체 유무 등으로 자동 판단
3. **커스터마이징**: 특수 환경에서는 `setIsServer()`로 판단 로직 재정의 가능
4. **전역 설정**: 한 번 설정하면 애플리케이션 전체에 적용됨

## 실전 팁

**왜 중요한가?**
TanStack Query는 서버와 클라이언트에서 다르게 동작합니다:
- **서버**: 데이터를 미리 가져와서(prefetch) HTML에 포함
- **클라이언트**: 캐싱, 자동 재검증, 백그라운드 업데이트 등 활성화

환경을 잘못 판단하면:
- 서버에서 불필요한 재검증 시도
- 클라이언트에서 SSR 기능 동작 안 함
- 메모리 누수나 성능 문제 발생 가능

**설정 타이밍**
앱 진입점(entry point)에서 가장 먼저 설정해야 합니다:
```tsx
// App.tsx 또는 index.tsx 최상단
import { environmentManager } from '@tanstack/react-query'

// React Query를 사용하기 전에 설정
environmentManager.setIsServer(() => {
  // 커스텀 로직
})
```
